# Clock Network Design

## Overview

The clock network distributes the timing reference to all sequential elements in the iPACE-CHIP pacemaker ASIC. For a medical device delivering precise pacing therapy, clock network design directly impacts timing accuracy, power consumption, and electromagnetic interference (EMI) performance.

## Clock Domains in iPACE-CHIP

### Domain Architecture

```
Primary Clock: CLK_EXT (32.768 kHz crystal oscillator)

Derived Clocks:
+-- CLK_CORE (100 MHz) - Digital logic
|   +-- Generated by PLL from CLK_EXT
|   +-- Duty cycle: 50%
|   +-- Jitter: < 50 ps RMS
|
+-- CLK_ANA (2 MHz) - Analog sampling
|   +-- Generated by divider from CLK_CORE
|   +-- Synchronous to CLK_CORE
|
+-- CLK_TELEM (1 MHz) - Telemetry communication
|   +-- Generated by divider from CLK_CORE
|   +-- Phase-aligned to CLK_CORE
|
+-- CLK_WKUP (32.768 kHz) - Wake-up timer
    +-- Direct from crystal oscillator
    + - Runs in sleep mode
```

### Clock Domain Crossing (CDC)

```tcl
# Define clock domains
create_clock -name CLK_CORE -period 10.0 -waveform {0 5.0} [get_ports clk_ext]
create_clock -name CLK_ANA -period 500.0 -waveform {0 250.0} [get_ports clk_analog]
create_clock -name CLK_TELEM -period 1000.0 -waveform {0 500.0} [get_ports clk_telem]

# Clock domain crossings
set_clock_groups -asynchronous \
    -group [get_clocks CLK_CORE] \
    -group [get_clocks CLK_ANA]

set_clock_groups -asynchronous \
    -group [get_clocks CLK_CORE] \
    -group [get_clocks CLK_TELEM]
```

## Clock Tree Architecture

### Tree Structure

```
                    CLK_EXT (32 kHz)
                         |
                    +----+----+
                    |   PLL   |
                    +----+----+
                         |
                    CLK_CORE (100 MHz)
                         |
              +----------+----------+
              |                     |
         +----+----+          +----+----+
         | Divider |          | Divider |
         +----+----+          +----+----+
              |                     |
         CLK_ANA (2 MHz)     CLK_TELEM (1 MHz)
              |                     |
    +---------+---------+   +------+------+
    |         |         |   |      |      |
  ADC1     ADC2      DAC  UART   SPI    I2C
```

### Clock Tree Buffer Strategy

```tcl
# Clock tree buffer library
# HVT buffers for low power, SVT for balanced, LVT for speed

set clock_buf_cells {
    CLKBUFX16HVT    ;# High drive, low leakage
    CLKBUFX8HVT     ;# Medium drive, low leakage
    CLKBUFX4HVT     ;# Low drive, low leakage
    CLKBUFX16SVT    ;# High drive, balanced
    CLKBUFX8SVT     ;# Medium drive, balanced
    CLKBUFX4SVT     ;# Low drive, balanced
    CLKINVX8SVT     ;# Inverter variant
    CLKAND2X4SVT    ;# Clock gating cell
    CKLNQD1         ;# Latch-based clock gate
}
```

## Clock Tree Synthesis Flow

### Innovus CTS Setup

```tcl
proc setup_clock_tree {} {
    # Step 1: Define clock tree specification
    create_clock_tree_spec -cell_list {CLKBUFX* CLKINV*}

    # Step 2: Set clock tree options
    setClockTreeOptions \
        -targetSkew 0.1 \
        -targetInsertionDelay 1.5 \
        -bufFootprint {CLKBUFX*} \
        -invFootprint {CLKINV*} \
        -rootBuffer {CLKBUFX16SVT} \
        -leafBuffer {CLKBUFX4SVT}

    # Step 3: Set routing rules
    setNanoRouteMode -routeWithClockNet true
    setNanoRouteMode -routeWithClockLayerOnly {M5 M6}

    # Step 4: Clock tree routing
    setClockTreeOptions -routeTopPreferredLayer M6 \
        -routeBottomPreferredLayer M5

    # Step 5: Set clock uncertainty
    set_clock_uncertainty 0.2 [get_clocks CLK_CORE]
    set_clock_transition 0.1 [get_clocks CLK_CORE]
}
```

## Clock Tree Buffer Sizing

### Driver Sizing Strategy

```python
# Clock tree buffer sizing for iPACE-CHIP
# Based on fanout and wire length

def size_clock_buffer(fanout, wire_length_um, target_slew):
    """Determine appropriate clock buffer size"""
    
    # Base sizes
    buffer_sizes = {
        'CLKBUFX4': {'drive': 4, 'area': 14.4, 'power': 2.1},
        'CLKBUFX8': {'drive': 8, 'area': 28.8, 'power': 4.2},
        'CLKBUFX16': {'drive': 16, 'area': 57.6, 'power': 8.4},
    }
    
    # Calculate required drive strength
    load_cap = fanout * 0.02 + wire_length_um * 0.0002  # pF
    required_drive = load_cap / (target_slew * 0.69)  # mA
    
    # Select buffer
    if required_drive < 4:
        return 'CLKBUFX4', buffer_sizes['CLKBUFX4']
    elif required_drive < 8:
        return 'CLKBUFX8', buffer_sizes['CLKBUFX8']
    else:
        return 'CLKBUFX16', buffer_sizes['CLKBUFX16']

# Example: Timing engine clock pin
# Fanout: 320, Wire length: 150 um
buf_name, buf_info = size_clock_buffer(320, 150, 0.15)
print(f"Selected: {buf_name}")
print(f"Area: {buf_info['area']} um^2, Power: {buf_info['power']} uW")
```

## Clock Tree Routing

### Dedicated Clock Routing Layers

```tcl
# Clock routing on M5/M6 only
setNanoRouteMode -routeWithClockLayerOnly {M5 M6}

# Clock routing width
setNetStat -net CLK_CORE -width 1.6 -layer M6
setNetStat -net CLK_ANA -width 1.2 -layer M5
setNetStat -net CLK_TELEM -width 0.8 -layer M5

# Clock routing rules
create_route_rule -name clock_rule \
    -topPreferredLayer M6 \
    -bottomPreferredLayer M5 \
    -width 1.6

setClockTreeOptions -routeRule clock_rule

# Route clock tree
routeDesign -clock
```

### Clock Shielding

```tcl
# Shield clock nets for EMI reduction
# iPACE-CHIP requires low EMI for implant safety

# Shield rules
create_route_rule -name clock_shield_rule \
    -width 1.6 \
    -spacing 2.0 \
    -shieldWidth 0.32 \
    -shieldSpacing 0.8

# Apply shielding to CLK_CORE
set_signal_net_shield -net CLK_CORE -shield_net VSS -bothSide

# Route with shielding
routeDesign -clock

# Verify shielding connectivity
verifyConnectivity -type regular -net {CLK_CORE CLK_shield}
```

## Clock Tree for Analog Domain

### Analog Clock Isolation

```tcl
# Separate clock routing for analog domain
# Prevents digital noise coupling into analog sampling clock

# Dedicated analog clock buffer
placeInst CLKBUFX4SVT_ana_buf 620 480 0

# Route analog clock on dedicated layer
setNetStat -net CLK_ANA -preferredLayer M5
setNanoRouteMode -routeWithClockLayerOnly {M5}

# Add guard ring around analog clock routing
addRing -nets {VSS} -type block_rings \
    -around {heart_signal_adc impedance_adc} \
    -layer {M2 M3} -width {3.0 3.0} -spacing {2.0 2.0}

# No digital signal routing within 20 um of analog clock
createRouteBlk -box {600 0 980 500} -layer {M3 M4}
```

## Summary

Clock network design for iPACE-CHIP uses a three-domain architecture (100 MHz core, 2 MHz analog, 1 MHz telemetry) with dedicated routing layers M5/M6. The clock tree employs 156 buffers and 128 clock gates, achieving 0.08 ns skew and 81 uW total power. Analog clock isolation and EMI shielding ensure the clock distribution does not interfere with sensitive cardiac signal measurement.
