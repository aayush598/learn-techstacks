# Netlists and Hierarchies in Verilog

## 5.2.4 — Overview

A **netlist** describes the complete connectivity of a digital design — every
module, every instance, every wire. A **hierarchy** organizes modules into a
tree structure. For iPACE-CHIP, the hierarchy flows from the digital top level
down through subsystem controllers to individual standard cells. This chapter
covers netlist representation, hierarchical design methodology, and best
practices for managing complex ASIC hierarchies.

---

### Module Hierarchy Levels

| Level | Description | Example |
|-------|-------------|---------|
| Top | Full chip | `ipace_digital_top` |
| Subsystem | Major functional block | `pacing_controller` |
| Module | Specific function | `pacing_timer` |
| Cell | Standard cell / gate | `SDFFARX1` |
| Transistor | MOSFET | `nmos`, `pmos` |

## 5.2.6 — Structural Netlist Description

### Flat Netlist

A flat netlist has no hierarchy — all instances at the top level:

```verilog
// Simplified flat netlist (after synthesis flattening)
module ipace_flat_netlist (
    input  wire clk,
    input  wire rst_n,
    input  wire [11:0] adc_data,
    output wire pace_out
);
    // All gates at top level (no hierarchy)
    wire w1, w2, w3, w4, w5;

    SDFFARX1 uff0 (.D(w1), .CK(clk), .RB(rst_n), .Q(w2));
    SDFFARX1 uff1 (.D(w3), .CK(clk), .RB(rst_n), .Q(w4));
    AO21X1   u01  (.A(adc_data[0]), .B(w2), .C(w4), .Q(w5));
    INVX1    u02  (.A(w5), .Q(w1));
    BUFX1    u03  (.A(w4), .Q(pace_out));
endmodule
```

### Hierarchical Netlist

```verilog
// Hierarchical — preserves module structure
module ipace_digital_top (
    input  wire        clk,
    input  wire        rst_n,
    input  wire [11:0] adc_data,
    output wire        pace_out
);
    wire timer_done, sense_event;

    pacing_timer u_timer (
        .clk(clk), .rst_n(rst_n), .timer_done(timer_done)
    );

    sensing_unit u_sense (
        .clk(clk), .rst_n(rst_n),
        .adc_data(adc_data), .sense_event(sense_event)
    );

    pacing_output u_pace (
        .clk(clk), .rst_n(rst_n),
        .timer_done(timer_done), .sense_event(sense_event),
        .pace_out(pace_out)
    );
endmodule
```

## 5.2.7 — Hierarchical References in Testbenches

```verilog
module system_tb;
    reg clk, rst_n;
    wire [31:0] debug_signals;

    // DUT instantiation
    ipace_digital_top u_dut (
        .clk(clk), .rst_n(rst_n), ...
    );

    // Hierarchical signal access (simulation only)
    wire [15:0] timer_count = u_dut.u_timer.count;
    wire [1:0]  timer_state = u_dut.u_timer.state;
    wire [7:0]  adc_processed = u_dut.u_sense.adc_out;

    // Force/release (simulation only)
    initial begin
        #1000;
        force u_dut.u_sense.adc_data = 12'h800;
        #500;
        release u_dut.u_sense.adc_data;
    end

    // Monitor hierarchical signals
    initial begin
        $monitor("T=%0t timer=%0d state=%0d",
                 $time, timer_count, timer_state);
    end
endmodule
```

## 5.2.8 — Parameterized Hierarchy

### Shared Parameters Across Hierarchy

```verilog
// Top-level parameters propagate to all children
module ipace_digital_top #(
    parameter ADC_WIDTH   = 12,
    parameter TIMER_WIDTH = 16,
    parameter SPI_WIDTH   = 8
)(
    ...
);
    // Parameters automatically passed down
    sensing_adc_interface #(
        .WIDTH(ADC_WIDTH)
    ) u_sensing (...);

    pacing_timer #(
        .WIDTH(TIMER_WIDTH)
    ) u_timer (...);
endmodule
```

### Hierarchical Parameter Override

```verilog
module system_top;
    // Override ADC width for specific instance
    ipace_digital_top #(
        .ADC_WIDTH(14)  // 14-bit ADC variant
    ) u_chip (...);
endmodule
```

## 5.2.9 — Netlist Formats

### Verilog Netlist (Gate-Level)

```verilog
// Generated by synthesis tool
module pacing_timer (
    CK, RB, EN, DONE
);
    input CK, RB, EN;
    output DONE;

    wire n1, n2, n3, n4, n5;

    SDFFARX1 U1 (.CK(CK), .RB(RB), .D(n1), .Q(n2));
    SDFFARX1 U2 (.CK(CK), .RB(RB), .D(n3), .Q(n4));
    AO21X1   U3 (.A(n2), .B(n4), .C(n5), .Q(DONE));
    INVX1    U4 (.A(DONE), .Q(n5));
endmodule
```

### DEF (Design Exchange Format) — Connectivity

```
VERSION 5.8 ;
DESIGN pacing_controller ;
UNITS DISTANCE MICRONS 1000 ;
DIE AREA 0 0 50000 50000 ;

COMPONENTS 15 ;
  - u_timer pacing_timer + PLACED ( 5000 5000 ) N ;
  - u_ff0 SDFFARX1 + PLACED ( 10000 10000 ) N ;
  ...
END COMPONENTS ;

NETS 45 ;
  - net_clk ( u_timer/CK u_ff0/CK ) ;
  - net_rst ( u_timer/RB u_ff0/RB ) ;
  ...
END NETS ;
```

## 5.2.10 — Pacemaker Complete Hierarchical Netlist

```verilog
// ==========================================
// ipace_digital_top — Structural Description
// ==========================================
module ipace_digital_top #(
    parameter ADC_WIDTH      = 12,
    parameter TIMER_WIDTH    = 16,
    parameter TELEMETRY_BITS = 64
)(
    // System
    input  wire                    clk_32khz,
    input  wire                    rst_n,

    // Analog front-end
    input  wire [ADC_WIDTH-1:0]    adc_atrial,
    input  wire [ADC_WIDTH-1:0]    adc_ventricular,
    input  wire                    adc_valid,

    // Pacing output
    output wire                    atrial_pulse,
    output wire                    ventricular_pulse,

    // SPI
    input  wire                    spi_clk,
    input  wire                    spi_mosi,
    output wire                    spi_miso,
    input  wire                    spi_cs_n
);

    // === Internal wires ===
    wire        timer_expired_atrial;
    wire        timer_expired_ventricular;
    wire        atrial_sense;
    wire        vent_sense;
    wire        refractory_atrial;
    wire        refractory_ventricular;
    wire [7:0]  config_data;
    wire [7:0]  telemetry_tx;
    wire        telemetry_tx_valid;
    wire [7:0]  telemetry_rx;
    wire        telemetry_rx_valid;
    wire        low_battery;
    wire        power_good;
    wire        pacing_inhibit;

    // === Level 1: Subsystem Instantiation ===

    // Pacing Controller
    pacing_controller #(
        .TIMER_WIDTH(TIMER_WIDTH)
    ) u_pacing_ctrl (
        .clk                (clk_32khz),
        .rst_n              (rst_n),
        .timer_expired_a    (timer_expired_atrial),
        .timer_expired_v    (timer_expired_ventricular),
        .sense_a            (atrial_sense),
        .sense_v            (vent_sense),
        .refrac_a           (refractory_atrial),
        .refrac_v           (refractory_ventricular),
        .config             (config_data),
        .inhibit            (pacing_inhibit),
        .pulse_a            (atrial_pulse),
        .pulse_v            (ventricular_pulse)
    );

    // Sensing ADC Interface
    sensing_adc_interface #(
        .WIDTH(ADC_WIDTH)
    ) u_sensing (
        .clk            (clk_32khz),
        .rst_n          (rst_n),
        .adc_a          (adc_atrial),
        .adc_v          (adc_ventricular),
        .adc_valid      (adc_valid),
        .sense_a        (atrial_sense),
        .sense_v        (vent_sense),
        .refrac_a       (refractory_atrial),
        .refrac_v       (refractory_ventricular)
    );

    // Telemetry Controller
    telemetry_controller #(
        .DATA_WIDTH(8),
        .PACKET_WIDTH(TELEMETRY_BITS)
    ) u_telemetry (
        .clk            (clk_32khz),
        .rst_n          (rst_n),
        .spi_clk        (spi_clk),
        .spi_mosi       (spi_mosi),
        .spi_miso       (spi_miso),
        .spi_cs_n       (spi_cs_n),
        .tx_data        (telemetry_tx),
        .tx_valid       (telemetry_tx_valid),
        .rx_data        (telemetry_rx),
        .rx_valid       (telemetry_rx_valid)
    );

    // Power Controller
    power_controller u_power (
        .clk            (clk_32khz),
        .rst_n          (rst_n),
        .low_battery    (low_battery),
        .power_good     (power_good),
        .pacing_inhibit (pacing_inhibit)
    );

    // === Level 2: Configuration Registers ===
    config_registers u_config (
        .clk        (clk_32khz),
        .rst_n      (rst_n),
        .write_en   (telemetry_rx_valid),
        .write_data (telemetry_rx),
        .config_out (config_data)
    );

endmodule
```

## 5.2.12 — Best Practices

1. **Maintain clean hierarchy** — each module = one clear function
2. **Use parameters** for all configurable values across hierarchy
3. **Name instances consistently** — `u_` prefix, descriptive names
4. **Limit hierarchy depth** — synthesis tools work best with < 20 levels
5. **Document netlist format** — Verilog, DEF, EDIF, or proprietary
6. **Verify equivalence** — RTL vs. gate-level with formal tools
7. **Use hierarchical testbenches** — match DUT hierarchy for debug
8. **Partition by clock domain** — separate hierarchy for each domain
9. **Check for unconnected ports** — lint tools catch floating nets
10. **Version control netlists** — critical for tapeout traceability
