# Java Interview Questions and Answers

## Q1: What is Java?
**A:** Java is a high-level, class-based, object-oriented programming language designed by James Gosling at Sun Microsystems and first released in 1995. It follows the "write once, run anywhere" (WORA) principle, meaning compiled Java bytecode runs on any platform with a Java Virtual Machine (JVM). It is statically typed, garbage-collected, and widely used for enterprise, mobile (Android), and web applications.

## Q2: What are the key features of Java?
**A:** Key features include: platform independence (JVM bytecode), object-oriented (inheritance, polymorphism, encapsulation, abstraction), automatic garbage collection, strong memory management, multithreading support, rich standard library (Java API), security (bytecode verification, sandboxing), dynamic class loading, and a large ecosystem of frameworks and tools.

## Q3: Explain the JVM, JRE, and JDK.
**A:** JVM (Java Virtual Machine) executes Java bytecode and provides runtime environment. JRE (Java Runtime Environment) includes JVM + core libraries for running Java applications. JDK (Java Development Kit) includes JRE + development tools (compiler, debugger, javadoc). JDK is needed for development; JRE is sufficient for running applications.

## Q4: What is bytecode in Java?
**A:** Bytecode is the intermediate representation of Java code, generated by the `javac` compiler from `.java` source files into `.class` files. Bytecode is platform-independent and executed by the JVM, which translates it into native machine code via interpretation or Just-In-Time (JIT) compilation. This is the foundation of Java's "write once, run anywhere" capability.

## Q5: What is the difference between JDK, JRE, and JVM?
**A:** JVM is the runtime engine that executes bytecode. JRE = JVM + libraries needed to run Java applications. JDK = JRE + development tools (compiler, debugger, etc.). To develop Java programs you need JDK. To only run Java programs, JRE is sufficient. JVM is part of both.

## Q6: What are the primitive data types in Java?
**A:** Java has 8 primitive types: `byte` (8-bit, -128 to 127), `short` (16-bit), `int` (32-bit), `long` (64-bit), `float` (32-bit IEEE 754), `double` (64-bit IEEE 754), `char` (16-bit Unicode), and `boolean` (true/false). Primitives are stored directly in stack memory and are not objects. They have fixed sizes and default values.

## Q7: What is the difference between `int` and `Integer` in Java?
**A:** `int` is a primitive type stored directly in stack memory. `Integer` is a wrapper class that wraps an int value into an object, stored on the heap. `Integer` provides utility methods like `parseInt()`, `toString()`, and comparison methods. Autoboxing/unboxing automatically converts between them. `Integer` can be null; `int` cannot.

## Q8: Explain autoboxing and unboxing.
**A:** Autoboxing is the automatic conversion of primitive types to their corresponding wrapper classes (e.g., `int` to `Integer`) by the compiler. Unboxing is the reverse — wrapper to primitive. This happens automatically in assignments, method arguments, and collections. Example: `List<Integer> list = new ArrayList<>(); list.add(42);` — autoboxing converts `42` to `Integer.valueOf(42)`.

## Q9: What is the difference between `==` and `.equals()` in Java?
**A:** `==` compares object references (memory addresses) for objects, and compares values for primitives. `.equals()` is a method that compares object content (logical equality). The default implementation in `Object` uses `==`, so classes like `String`, `Integer` override `.equals()` to compare values. Always use `.equals()` for content comparison of objects.

## Q10: What is the `String` pool in Java?
**A:** The String pool is a special memory region in the heap where Java caches string literals. When a string literal is created (e.g., `String s = "hello"`), the JVM checks the pool first — if it exists, the reference is returned; otherwise, a new string is created and cached. This saves memory and improves performance. Strings created with `new String("hello")` bypass the pool unless `.intern()` is called.

## Q11: Why are strings immutable in Java?
**A:** Strings are immutable for several reasons: thread safety (immutable objects are inherently thread-safe), string pool caching (immutability guarantees cached strings won't change), security (class loading, database URLs, etc. use strings), performance (hash code caching), and preventing unintended modification. Any "modification" creates a new string.

## Q12: Explain `StringBuilder` vs `StringBuffer`.
**A:** Both are mutable sequences of characters. `StringBuffer` is thread-safe (synchronized methods) but slower. `StringBuilder` is not thread-safe but faster. In single-threaded scenarios, `StringBuilder` is preferred. Both are more efficient than string concatenation in loops. `StringBuilder` was introduced in Java 5 as a faster alternative.

## Q13: What is the difference between `final`, `finally`, and `finalize`?
**A:** `final` is a keyword for declarations: final classes can't be subclassed, final methods can't be overridden, final variables can't be reassigned. `finally` is a block in try-catch-finally that always executes (used for cleanup). `finalize()` is a method called by the garbage collector before reclaiming an object (deprecated in Java 9, removed in Java 18).

## Q14: What are access modifiers in Java?
**A:** Access modifiers control visibility: `public` (accessible from anywhere), `protected` (accessible within the same package and subclasses), default/package-private (accessible only within the same package), `private` (accessible only within the same class). They are used to implement encapsulation.

## Q15: Explain `static` keyword in Java.
**A:** `static` means the member belongs to the class rather than instances. Static variables are shared across all instances. Static methods can be called without creating an instance (they cannot access instance variables directly). Static blocks run when the class is first loaded. Static classes are only allowed as nested classes.

## Q16: What is the difference between instance variables and static variables?
**A:** Instance variables belong to individual objects — each object has its own copy, initialized when the object is created. Static variables belong to the class — all instances share the same copy, loaded when the class is loaded. Static variables can be accessed directly via the class name without creating an instance.

## Q17: Explain method overloading vs overriding.
**A:** Overloading: multiple methods with the same name but different parameters in the same class — compile-time polymorphism. Overriding: a subclass provides a specific implementation of a method from its superclass — runtime polymorphism. Overloaded methods have the same name but different signatures; overridden methods have the exact same signature.

## Q18: Can you override a private or static method?
**A:** Private methods cannot be overridden — they are not visible to subclasses. If a subclass defines a method with the same name, it's a new method, not an override. Static methods cannot be overridden either — they are class-level and method resolution happens at compile-time based on the reference type (method hiding, not overriding).

## Q19: What is the difference between abstract class and interface?
**A:** Abstract classes can have both abstract and concrete methods, instance variables, constructors, and any access modifiers. Interfaces (pre-Java 8) had only abstract methods and constants. Java 8+ interfaces support default and static methods. Java 9+ supports private methods in interfaces. A class can extend only one abstract class but implement multiple interfaces. Abstract classes are for "is-a" relationships with shared state; interfaces define contracts/capabilities.

## Q20: What is the `default` method in interfaces?
**A:** Introduced in Java 8, `default` methods allow interfaces to provide method implementations. Classes implementing the interface can use the default implementation or override it. This was introduced primarily to add methods to existing interfaces without breaking implementing classes (e.g., adding `stream()` to `Collection`).

## Q21: Explain the diamond problem in Java.
**A:** The diamond problem occurs when a class implements two interfaces that have default methods with the same signature. Java resolves this by requiring the implementing class to override the conflicting method explicitly using `super` to specify which interface's implementation to use: `InterfaceName.super.methodName()`. Java avoids the classic diamond problem of multiple inheritance by not allowing multiple class inheritance.

## Q22: What is inheritance in Java?
**A:** Inheritance is a mechanism where a class (subclass/child) acquires properties and behaviors of another class (superclass/parent) using the `extends` keyword. Java supports single inheritance (a class can extend only one superclass) but supports multilevel inheritance. Inheritance promotes code reuse, method overriding, and polymorphic behavior.

## Q23: What is polymorphism in Java?
**A:** Polymorphism means "many forms" — the ability of an object to take multiple forms. Compile-time polymorphism (method overloading) — the method to call is determined at compile time. Runtime polymorphism (method overriding) — the method to call is determined at runtime based on the actual object type, not the reference type. Achieved through inheritance and interface implementation.

## Q24: What is encapsulation in Java?
**A:** Encapsulation is the bundling of data (variables) and methods that operate on that data into a single unit (class), and restricting direct access to internal details using access modifiers. Data is accessed only through public getter/setter methods. This protects object integrity, enables validation, and allows internal implementation changes without affecting external code.

## Q25: What is the `super` keyword?
**A:** `super` refers to the immediate parent class object. It's used to: access parent class fields (when shadowed), call parent class methods (when overridden), and invoke parent class constructor (`super()`) — which must be the first statement in the subclass constructor. If not explicitly called, the compiler inserts `super()` automatically.

## Q26: What is the difference between `this` and `super`?
**A:** `this` refers to the current class instance, while `super` refers to the immediate parent class instance. `this()` calls another constructor in the same class; `super()` calls the parent constructor. `this` can be used to access all members of the current class; `super` accesses only inherited members.

## Q27: What is a constructor in Java?
**A:** A constructor is a special method that initializes objects when they are created using `new`. It has the same name as the class and no return type. Java provides a default no-arg constructor if none is defined. Constructors can be overloaded. They are not inherited. Constructors can call other constructors via `this()` or parent constructors via `super()`.

## Q28: Can a constructor be private?
**A:** Yes, a private constructor prevents instantiation from outside the class. This is used in: singleton pattern (single instance), utility classes (static methods only, e.g., `Math`, `Collections`), factory methods, and builder patterns. Private constructors don't prevent the class from being instantiated internally.

## Q29: What is a copy constructor in Java?
**A:** A copy constructor creates a new object by copying another object of the same class. Unlike C++, Java doesn't provide a default copy constructor — you must define it explicitly. Example: `public Person(Person other) { this.name = other.name; }`. It's often preferred over `clone()` for copying objects.

## Q30: Explain the `Object` class methods.
**A:** `Object` is the root class of all Java classes. Key methods: `equals(Object)` — compares object equality; `hashCode()` — returns hash code for hashing; `toString()` — string representation; `clone()` — creates a shallow copy; `finalize()` — deprecated cleanup; `getClass()` — runtime class info; `notify()`, `notifyAll()`, `wait()` — thread synchronization.

## Q31: What is the contract between `equals()` and `hashCode()`?
**A:** The contract states: if two objects are equal according to `equals()`, they must have the same `hashCode()`. The reverse is not required (different objects can share a hash code). Violating this contract means hash-based collections (`HashMap`, `HashSet`, `HashTable`) will behave incorrectly — objects may be lost or unreachable. Always override both methods together.

## Q32: How do you properly override `equals()`?
**A:** The steps: use `==` to check if same reference; use `instanceof` to check correct type; cast to the correct type; compare each significant field. Rules: reflexive (`x.equals(x)` true), symmetric (`x.equals(y)` = `y.equals(x)`), transitive, consistent (no side effects), and non-null (`x.equals(null)` false).

## Q33: What is the `hashCode()` method used for?
**A:** `hashCode()` returns an integer hash code used by hash-based collections like `HashMap`, `HashSet`, and `HashTable` to determine the bucket for storing an object. A good hash function distributes objects uniformly across buckets for O(1) performance. The default implementation in `Object` returns the memory address, which doesn't work with overridden `equals()`.

## Q34: What is the difference between `ArrayList` and `LinkedList`?
**A:** `ArrayList` uses a dynamic array — O(1) random access, O(n) insertion/removal in the middle, fast iteration, memory efficient (only stores data). `LinkedList` uses a doubly-linked list — O(n) random access, O(1) insertion/removal at ends, extra memory for node pointers. Use `ArrayList` for most cases unless frequent add/remove at the beginning is needed.

## Q35: What is the difference between `HashMap` and `Hashtable`?
**A:** `HashMap` is not synchronized (not thread-safe), allows one null key and multiple null values, and is generally preferred. `Hashtable` is synchronized (thread-safe), does not allow null keys or values, and is considered legacy. `ConcurrentHashMap` is the modern thread-safe alternative to both.

## Q36: How does `HashMap` work internally?
**A:** `HashMap` stores key-value pairs in an array of `Node<K,V>` (buckets). It uses the key's `hashCode()` to compute the bucket index. If multiple keys hash to the same bucket (collision), entries are stored as a linked list (or balanced tree if the list exceeds 8 entries — TREEIFY_THRESHOLD). On `put`, it checks existing keys by `equals()`; on `get`, it locates the bucket and searches linearly for the matching key.

## Q37: What is the load factor in `HashMap`?
**A:** The load factor (default 0.75) determines when the `HashMap` resizes (rehashes). When the number of entries exceeds `capacity * load factor`, the capacity is doubled and all entries are rehashed. A higher load factor (0.9) saves memory but increases lookup time; a lower factor (0.5) uses more memory but reduces collisions. 0.75 balances time and space.

## Q38: Explain `ConcurrentHashMap` internals.
**A:** `ConcurrentHashMap` allows concurrent reads and writes without locking the entire map. In Java 8+: it uses a Node array with CAS (Compare-And-Swap) for insertion and `synchronized` on individual bucket heads for updates. The `size()` method uses `sumCount()` which aggregates values maintained in `CounterCell` array. It divides the map into segments/buckets for fine-grained locking.

## Q39: What is the difference between `Set` and `Map`?
**A:** A `Set` is a collection of unique elements (no duplicates), modeled after mathematical sets. A `Map` stores key-value pairs with unique keys. `Set` extends `Collection`; `Map` does not. Common implementations: `HashSet` (backed by `HashMap`), `TreeSet` (sorted, backed by `TreeMap`), `LinkedHashSet` (insertion order).

## Q40: What is the difference between `HashSet` and `TreeSet`?
**A:** `HashSet` uses a hash table — O(1) operations, no ordering, allows null. `TreeSet` uses a Red-Black tree — O(log n) operations, elements sorted by natural order or a `Comparator`, does not allow null (comparison fails). `LinkedHashSet` is between them — hash-based with insertion-order iteration.

## Q41: What is the difference between `Comparable` and `Comparator`?
**A:** `Comparable` defines natural ordering within the class itself (`compareTo` method) — used when a class has a single natural comparison order. `Comparator` is a separate strategy object that defines custom ordering — used when multiple sort orders are needed or when you can't modify the class. `Comparable` is in `java.lang`; `Comparator` is in `java.util`.

## Q42: How do you sort a list of custom objects?
**A:** Either implement `Comparable` in the class and use `Collections.sort(list)` or `list.sort(null)`, or pass a `Comparator` to `Collections.sort(list, comparator)` or `list.sort(comparator)`. Java 8+ also supports `list.stream().sorted(Comparator.comparing(CustomObject::getProperty))`. The class must implement `Comparable` for natural sorting.

## Q43: What are generics in Java?
**A:** Generics (introduced in Java 5) enable type-safe collections and classes by allowing types to be parameters. Example: `List<String>` ensures only `String` objects are added, catching type errors at compile time. Generics eliminate explicit casting. They work with classes, methods, and interfaces. Under the hood, type erasure removes generic type information at runtime.

## Q44: What is type erasure in generics?
**A:** Type erasure is the process where the compiler removes generic type information and replaces type parameters with their bounds (or `Object` if unbounded) during compilation. This ensures backward compatibility with pre-generics code. For example, `List<String>` becomes `List` at runtime. Bridge methods are generated to preserve polymorphism.

## Q45: What are bounded type parameters?
**A:** Bounded type parameters restrict the types that can be used as type arguments. Upper bound: `<T extends Number>` — T can be Number or any subclass. Lower bound: `<? super Integer>` — wildcard that accepts Integer or its superclasses. Multiple bounds: `<T extends A & B & C>` — T must extend all specified types (class first, then interfaces).

## Q46: What is the difference between `? extends T` and `? super T`?
**A:** `? extends T` (upper-bounded wildcard) — you can READ from the collection (producer). You cannot add elements (except null) because the exact type is unknown. `? super T` (lower-bounded wildcard) — you can WRITE T or subtypes (consumer). Reading returns `Object`. PECS: Producer Extends, Consumer Super.

## Q47: Explain the PECS principle.
**A:** PECS stands for "Producer Extends, Consumer Super." If a parameter produces values (you read from it), use `? extends T`. If a parameter consumes values (you write to it), use `? super T`. Neither if both (use exact type). This is the rule for using wildcards in Java generics, derived from `Collection.addAll()` and `Collections.copy()`.

## Q48: What are checked and unchecked exceptions?
**A:** Checked exceptions are checked at compile time — the compiler forces the caller to handle or declare them (`IOException`, `SQLException`). They extend `Exception` but not `RuntimeException`. Unchecked exceptions are not checked at compile time — they extend `RuntimeException` or `Error` (`NullPointerException`, `ArrayIndexOutOfBoundsException`). Checked exceptions represent recoverable conditions; unchecked represent programming errors.

## Q49: What is the difference between `throw` and `throws`?
**A:** `throw` is used to explicitly throw an exception from a method or block: `throw new IOException("error")`. `throws` is used in method signatures to declare that a method might throw certain checked exceptions: `public void readFile() throws IOException`. A method can `throw` multiple exceptions but `throws` declares the exception types.

## Q50: What is the try-with-resources statement?
**A:** Introduced in Java 7, try-with-resources automatically closes resources implementing `AutoCloseable` (or `Closeable`). Syntax: `try (FileReader fr = new FileReader("file.txt"); BufferedReader br = new BufferedReader(fr)) { ... }`. Resources are closed in reverse order automatically, even if an exception occurs. Multiple resources can be declared, separated by semicolons.

## Q51: What is a custom exception and how do you create one?
**A:** Create a class extending `Exception` (checked) or `RuntimeException` (unchecked). Provide constructors: default, with message, with cause, with both. Example: `public class InvalidAgeException extends RuntimeException { public InvalidAgeException(String message) { super(message); } }`. Custom exceptions make code more readable by conveying specific error semantics.

## Q52: Explain the difference between `Error` and `Exception`.
**A:** Both extend `Throwable`. `Error` represents serious, unrecoverable problems that should not be caught (`OutOfMemoryError`, `StackOverflowError`, `VirtualMachineError`) — typically JVM-level issues. `Exception` represents conditions that a program might want to catch and handle. `RuntimeException` (unchecked) represents programming bugs. `Error` and `RuntimeException` are unchecked; other `Exception` subclasses are checked.

## Q53: What is multi-catch in Java?
**A:** Introduced in Java 7, multi-catch allows catching multiple exception types in a single `catch` block using `|`: `catch (IOException | SQLException e) { ... }`. The caught exception is implicitly final. This reduces code duplication when handling different exceptions with the same logic. The exception types must not be in a parent-child relationship.

## Q54: What is the `Optional` class?
**A:** Introduced in Java 8, `Optional<T>` is a container that may or may not contain a non-null value. It helps avoid `NullPointerException` by explicitly indicating that a value might be absent. Methods: `of()`, `ofNullable()`, `empty()`, `isPresent()`, `ifPresent()`, `orElse()`, `orElseGet()`, `orElseThrow()`, `map()`, `flatMap()`, `filter()`. It's not meant for serialization or as a field type.

## Q55: What is the Stream API in Java?
**A:** Introduced in Java 8, the Stream API enables functional-style operations on collections. Streams represent a sequence of elements supporting sequential and parallel aggregate operations. Operations are: intermediate (filter, map, sorted, distinct, limit, skip) — return a new stream; terminal (forEach, collect, reduce, count, anyMatch) — produce a result. Streams don't modify the source.

## Q56: What is the difference between intermediate and terminal operations?
**A:** Intermediate operations transform a stream into another stream and are lazy — they don't execute until a terminal operation is invoked. Examples: `filter()`, `map()`, `flatMap()`, `sorted()`, `distinct()`, `peek()`, `limit()`, `skip()`. Terminal operations produce a result or side effect and consume the stream. Examples: `forEach()`, `collect()`, `reduce()`, `count()`, `anyMatch()`, `findFirst()`. A stream can only be consumed once.

## Q57: What is `map` vs `flatMap` in streams?
**A:** `map()` applies a function to each element and returns a stream of results — a one-to-one mapping. `flatMap()` applies a function that returns a stream for each element, then flattens the results into a single stream — a one-to-many mapping. Example: `flatMap(s -> s.chars().mapToObj(c -> (char) c)` flattens strings into character streams.

## Q58: Explain `reduce` in streams.
**A:** `reduce()` performs a reduction on stream elements using an associative accumulation function. Three forms: `reduce(BinaryOperator<T>)` returns `Optional<T>`; `reduce(T identity, BinaryOperator<T>)` returns `T` with an identity value (e.g., 0 for sum); `reduce(U identity, BiFunction<U,T,U>, BinaryOperator<U>)` for parallel reduction with result type conversion. Example: `numbers.stream().reduce(0, Integer::sum)`.

## Q59: What is `collect()` and `Collectors` in streams?
**A:** `collect()` is a terminal operation that accumulates stream elements into a container. `Collectors` provides implementations: `toList()`, `toSet()`, `toMap()`, `joining()`, `groupingBy()`, `partitioningBy()`, `summarizingInt()`, `counting()`, `mapping()`, `reducing()`. Example: `list.stream().collect(Collectors.groupingBy(Person::getCity, Collectors.counting()))`.

## Q60: Explain parallel streams.
**A:** Parallel streams leverage multiple CPU cores by splitting the stream into substreams processed concurrently and then combining results. Created via `collection.parallelStream()` or `stream.parallel()`. Under the hood, they use the Fork-Join pool. Best for CPU-intensive operations on large datasets with stateless, independent operations. May degrade performance for small datasets or I/O-bound tasks.

## Q61: What are lambda expressions?
**A:** Lambdas (Java 8+) provide a concise syntax for implementing functional interfaces using `(parameters) -> expression` or `(parameters) -> { statements; }`. They enable functional programming. Example: `(a, b) -> a + b`. Lambdas can access effectively-final local variables and instance/static variables. They replace anonymous inner classes for single-method interfaces.

## Q62: What is a functional interface?
**A:** A functional interface has exactly one abstract method (SAM — Single Abstract Method). It can have multiple default or static methods. The `@FunctionalInterface` annotation is optional but enforces the contract. Examples: `Runnable`, `Callable`, `Comparator`, `Consumer<T>`, `Supplier<T>`, `Function<T,R>`, `Predicate<T>`. Lambdas can be assigned to functional interface types.

## Q63: What are the main functional interfaces in `java.util.function`?
**A:** Key interfaces: `Consumer<T>` (accepts T, returns void), `Supplier<T>` (returns T), `Function<T,R>` (maps T to R), `Predicate<T>` (tests T, returns boolean), `UnaryOperator<T>` (T → T), `BinaryOperator<T>` (T,T → T). Primitive variants avoid boxing: `IntConsumer`, `DoubleSupplier`, `ToIntFunction`, `IntPredicate`, etc.

## Q64: What is a method reference?
**A:** Method references are shorthand for lambdas calling a specific method. Syntax: `ClassName::methodName` or `object::methodName`. Types: static method (`Integer::parseInt`), instance method of a particular object (`list::add`), instance method of an arbitrary object (`String::length`), constructor (`ArrayList::new`). Method references make code more concise and readable.

## Q65: What is the difference between `Stream` and `IntStream`?
**A:** `Stream<T>` is a generic stream for object types. `IntStream` is a specialized stream for primitive `int` values — it avoids boxing/unboxing overhead and provides type-specific operations like `sum()`, `average()`, `range()`, `rangeClosed()`, `boxed()`, `asDoubleStream()`, `mapToObj()`. Similar specialized streams exist for `LongStream` and `DoubleStream`.

## Q66: What is the Fork-Join framework?
**A:** Introduced in Java 7, the Fork-Join framework is designed for parallel processing using a divide-and-conquer approach. It uses `ForkJoinPool` (a work-stealing thread pool) and `RecursiveTask<V>` (returns result) or `RecursiveAction` (void). Tasks split (fork) subtasks until small enough to process directly, then combine (join) results. Used by parallel streams.

## Q67: What is the difference between `Runnable` and `Callable`?
**A:** `Runnable` represents a task that returns no result and cannot throw checked exceptions. Its method is `run()`. `Callable<V>` represents a task that returns a result of type V and can throw checked exceptions. Its method is `call()`. `Callable` is typically used with `ExecutorService` methods like `submit()` returning `Future<V>`.

## Q68: What is `Future` and `CompletableFuture`?
**A:** `Future` (Java 5) represents the result of an asynchronous computation with methods: `get()` (blocking), `cancel()`, `isDone()`, `isCancelled()`. `CompletableFuture` (Java 8) extends `Future` and provides: non-blocking completion callbacks (`thenApply`, `thenAccept`, `thenRun`), combining futures (`thenCombine`, `allOf`, `anyOf`), exception handling (`exceptionally`, `handle`), and explicit completion. It supports functional composition of async operations.

## Q69: Explain `synchronized` keyword.
**A:** `synchronized` ensures mutual exclusion — only one thread can execute the synchronized block/method at a time. It can be applied to: instance methods (locks on `this`), static methods (locks on the `Class` object), or arbitrary code blocks (locks on a specified object). Synchronization guarantees visibility (changes by one thread are visible to others) and atomicity.

## Q70: What is the difference between `synchronized` and `Lock`?
**A:** `synchronized` is built-in, automatic lock release on exception, simpler syntax. `Lock` (`ReentrantLock`) from `java.util.concurrent.locks` provides: try-lock (non-blocking), timed lock, interruptible lock, fair locking, multiple condition variables (`Condition`). `synchronized` uses intrinsic locking; `Lock` uses explicit locking requiring manual `lock()` and `unlock()` calls.

## Q71: What is `volatile` in Java?
**A:** `volatile` is a modifier that ensures visibility of changes across threads. Reads and writes to a `volatile` variable happen directly from main memory (not thread cache), guaranteeing that one thread's write is immediately visible to other threads. It also establishes a happens-before relationship. It doesn't provide atomicity (use `AtomicInteger` for compound operations).

## Q72: What is `AtomicInteger` and why use it?
**A:** `AtomicInteger` provides thread-safe operations on integers without synchronization, using low-level CAS (Compare-And-Swap) instructions at the hardware level. Methods: `incrementAndGet()`, `decrementAndGet()`, `addAndGet()`, `compareAndSet()`, `getAndSet()`. It's lock-free, non-blocking, and generally faster than `synchronized` for simple atomic operations. Similar classes: `AtomicLong`, `AtomicBoolean`, `AtomicReference`.

## Q73: What is the difference between `Thread` and `Runnable`?
**A:** `Thread` is a class that represents a thread of execution. `Runnable` is a functional interface that defines the task to execute. Extending `Thread` locks you into its hierarchy; implementing `Runnable` leaves your class free to extend other classes. `Runnable` is preferred for task separation and reusability.

## Q74: Explain thread pool in Java.
**A:** A thread pool manages a pool of worker threads, reusing them for multiple tasks to avoid thread creation overhead. `ExecutorService` is the main interface. Factory methods in `Executors`: `newFixedThreadPool(n)` (fixed size), `newCachedThreadPool()` (dynamic), `newSingleThreadExecutor()` (single thread), `newScheduledThreadPool(n)` (scheduled/delayed tasks). `ThreadPoolExecutor` provides full configuration.

## Q75: What is `CyclicBarrier` vs `CountDownLatch`?
**A:** `CountDownLatch` is a one-time barrier — threads wait until a count reaches zero (cannot be reset). Used for "wait until N operations complete." `CyclicBarrier` is reusable — threads wait until all N threads arrive at the barrier point, then they proceed together. It can be reset via `reset()`. Used for "all threads meet at a point" patterns.

## Q76: What is the `java.time` package?
**A:** Introduced in Java 8, `java.time` provides a comprehensive date-time API based on Joda-Time. Key classes: `LocalDate` (date without time), `LocalTime` (time without date), `LocalDateTime` (date and time), `ZonedDateTime` (date/time with timezone), `Instant` (machine timestamp), `Duration` (time-based amount), `Period` (date-based amount), `DateTimeFormatter` (parsing/formatting). It's immutable and thread-safe.

## Q77: What is the difference between `LocalDate`, `LocalTime`, and `LocalDateTime`?
**A:** `LocalDate` represents a date (year, month, day) without time or timezone. `LocalTime` represents a time (hour, minute, second, nanosecond) without date or timezone. `LocalDateTime` combines date and time. All are immutable and thread-safe. For timezone-aware operations, use `ZonedDateTime` or `OffsetDateTime`.

## Q78: What is `Instant` in Java?
**A:** `Instant` represents a specific moment on the timeline in UTC (machine-readable timestamp). It's stored as seconds since Unix epoch (1970-01-01T00:00:00Z) plus nanoseconds. Useful for timestamps in logs, timers, and system boundaries. Convert to/from `LocalDateTime` via timezone: `instant.atZone(ZoneId.of("UTC")).toLocalDateTime()`.

## Q79: What are annotations in Java?
**A:** Annotations are metadata tags attached to code elements (classes, methods, fields). They start with `@`. Built-in annotations: `@Override`, `@Deprecated`, `@SuppressWarnings`, `@FunctionalInterface`. Meta-annotations: `@Retention` (SOURCE/CLASS/RUNTIME), `@Target` (where annotation applies), `@Inherited`, `@Documented`, `@Repeatable`. Custom annotations are defined with `@interface`.

## Q80: What is reflection in Java?
**A:** Reflection allows inspecting and manipulating classes, methods, fields, and constructors at runtime using the `java.lang.reflect` package. Key classes: `Class`, `Method`, `Field`, `Constructor`. Uses: frameworks (Spring, Hibernate), serialization, testing, IDE code completion, dependency injection. Reflection is powerful but slow, bypasses access control, and is not type-safe.

## Q81: What are the `Class.forName()` and `newInstance()` methods?
**A:** `Class.forName("com.example.MyClass")` loads the class dynamically and returns the `Class` object. `clazz.newInstance()` (deprecated in Java 9) creates a new instance using the no-arg constructor. Modern alternative: `clazz.getDeclaredConstructor().newInstance()`. Used in JDBC driver loading (`Class.forName("com.mysql.jdbc.Driver")`) and frameworks.

## Q82: What is the `ClassLoader` in Java?
**A:** The ClassLoader loads Java classes into the JVM at runtime. Three built-in class loaders: Bootstrap (core Java libraries, `rt.jar`), Extension (ext libraries, `lib/ext`), Application/System (classpath). They follow the delegation model — a class loader first delegates to its parent before attempting to load the class. Custom class loaders can load classes from databases, networks, or encrypted sources.

## Q83: What is the difference between `static` and `dynamic` class loading?
**A:** Static loading: classes are loaded when the JVM starts using `import` and the class is referenced in code. Dynamic loading: classes are loaded at runtime using `Class.forName()`, `ClassLoader.loadClass()`, or reflection. Dynamic loading enables lazy loading, plugins, and frameworks to load classes without compile-time dependencies.

## Q84: What is `Serializable` in Java?
**A:** `Serializable` is a marker interface (no methods) that enables object serialization — converting an object's state to a byte stream for persistence, network transmission, or caching. The `ObjectOutputStream` writes serialized objects; `ObjectInputStream` reads them. Fields marked `transient` are skipped during serialization. `serialVersionUID` ensures class version compatibility during deserialization.

## Q85: What is the purpose of `serialVersionUID`?
**A:** `serialVersionUID` is a unique identifier for each serializable class, used during deserialization to verify that the sender and receiver have compatible class versions. If not declared explicitly, the JVM generates one based on class structure — which changes when the class is modified, potentially causing `InvalidClassException`. Always declare it explicitly to maintain compatibility.

## Q86: What is the difference between `Serializable` and `Externalizable`?
**A:** `Serializable` is a marker interface — Java handles serialization automatically (all non-transient fields). `Externalizable` extends `Serializable` with `writeExternal()` and `readExternal()` methods — giving the class complete control over serialization. `Externalizable` is more complex but can be more efficient and handle non-serializable objects.

## Q87: What is the `transient` keyword?
**A:** `transient` marks a field as not part of the serialized state. When an object is serialized, transient fields are skipped and their default values (`null` for objects, 0 for numbers) are used during deserialization. Used for: sensitive data (passwords), derived/cached data, non-serializable fields, or data that can be reconstructed.

## Q88: What is garbage collection in Java?
**A:** Garbage Collection (GC) is the automatic process of reclaiming memory occupied by objects that are no longer reachable. The JVM's GC runs in the background, identifying unreachable objects and freeing their memory. Heap is divided into generations: Young Generation (Eden, S0, S1) for new objects, Old Generation for long-lived objects, and Metaspace for class metadata.

## Q89: Explain the different garbage collectors in Java.
**A:** Major GC algorithms: Serial GC (single-threaded, for small apps), Parallel GC (multi-threaded, throughput-oriented, default in Java 8), G1 GC (low-pause, region-based, default since Java 9), ZGC (ultra-low latency, handles TB-sized heaps, available since Java 11), Shenandoah (low-pause, concurrent compaction). Each balances throughput, latency, and memory overhead.

## Q90: What is the difference between Minor GC, Major GC, and Full GC?
**A:** Minor GC collects the Young Generation — frequent, fast, stops-the-world briefly. Major GC collects the Old Generation. Full GC collects the entire heap (Young + Old + Metaspace) — long pause, often triggered by promotion failures, `System.gc()`, or metaspace threshold. Modern collectors like G1 minimize Full GC pauses.

## Q91: What are the JVM memory areas?
**A:** Heap: stores all objects and arrays — shared across threads. Stack: stores method frames (local variables, partial results) — one per thread. Method Area / Metaspace: stores class structures, static variables, constants. Program Counter Register: stores current instruction address per thread. Native Method Stack: supports native methods. Heap is the largest and GC-managed area.

## Q92: What is the `-Xms` and `-Xmx` JVM parameters?
**A:** `-Xms` sets the initial heap size (e.g., `-Xms512m`). `-Xmx` sets the maximum heap size (e.g., `-Xmx4g`). Setting equal values prevents heap resizing overhead. The JVM starts with `-Xms` and can grow up to `-Xmx`. Other memory params: `-XX:MaxMetaspaceSize`, `-Xss` (stack size per thread), `-XX:NewRatio` (young/old ratio).

## Q93: What is the `finalize()` method and why is it deprecated?
**A:** `finalize()` was called by the GC before reclaiming an object's memory. It's deprecated since Java 9 and removed in Java 18 because: execution timing is unpredictable, it may never run, it can resurrect objects, and it negatively impacts GC performance. Replace with try-with-resources (for resources) or `Cleaner`/`PhantomReference` (for native memory).

## Q94: What is the difference between `Heap` and `Stack` memory?
**A:** Heap: stores all objects and arrays, shared across threads, managed by GC, slower access, thread-safe via synchronization. Stack: stores local variables, method call frames, per-thread, automatically deallocated when method exits, faster access, inherently thread-safe (each thread has its own stack). Stack size is typically smaller than heap.

## Q95: What is Java Memory Model (JMM)?
**A:** JMM defines how threads interact through memory, specifying: happens-before relationships, visibility guarantees, atomicity rules, and reordering restrictions. It ensures: unlock on a monitor happens-before every subsequent lock, write to a volatile field happens-before every subsequent read, thread `start()` happens-before its actions, and thread's actions happen-before `join()` returns.

## Q96: What is the `module` system in Java 9 (Project Jigsaw)?
**A:** The module system (JPMS) introduces modularization of the JDK and applications. A module is a named, self-describing collection of code and data, declared in `module-info.java`. Key features: `requires` (module dependencies), `exports` (packages visible to other modules), `opens` (reflection access), `provides...with` (service provider), `uses` (service consumer). Improves encapsulation and reduces runtime footprint.

## Q97: What are `var` and local variable type inference?
**A:** Introduced in Java 10, `var` allows the compiler to infer the type of local variables from the initializer: `var list = new ArrayList<String>();` is equivalent to `List<String> list = new ArrayList<>();`. `var` can only be used for local variables with initializers (not fields, method parameters, or return types). It cannot be used without initialization.

## Q98: What are Records in Java?
**A:** Records (Java 16, preview in Java 14) are transparent carriers for immutable data. A record `record Point(int x, int y) {}` automatically generates: canonical constructor, accessor methods (`x()`, `y()`), `equals()`, `hashCode()`, `toString()`. Records are implicitly `final`, extend `java.lang.Record`, and cannot be extended. Components are `private final` fields.

## Q99: What are Sealed classes in Java?
**A:** Sealed classes (Java 17) restrict which classes can extend or implement them. Syntax: `sealed class Shape permits Circle, Rectangle, Triangle { }`. Permitted subclasses must be `final`, `sealed`, or `non-sealed`. This enables exhaustive pattern matching and gives more control over class hierarchies. Sealed interfaces follow the same pattern.

## Q100: What are Pattern Matching and Switch Expressions in Java?
**A:** Pattern Matching for `instanceof` (Java 16): `if (obj instanceof String s) { System.out.println(s.length()); }` — combines type check and variable declaration. Switch Expressions (Java 14): `int result = switch (day) { case MONDAY -> 1; case FRIDAY -> 2; default -> 0; };` — returns values, no fall-through. Switch Pattern Matching (Java 17+): `switch (obj) { case String s -> ...; case Integer i -> ...; default -> ...; }`.
