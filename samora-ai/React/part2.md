# React Interview Questions and Answers - Part 2

## Q1: How do you implement infinite scrolling in React?
**A:** Use the Intersection Observer API with a custom hook: `function useInfiniteScroll(callback) { const observer = useRef(); const lastElementRef = useCallback((node) => { if (observer.current) observer.current.disconnect(); observer.current = new IntersectionObserver(entries => { if (entries[0].isIntersecting) callback(); }); if (node) observer.current.observe(node); }, [callback]); return lastElementRef }`. Attach the ref to a sentinel element at the bottom of the list. When it becomes visible, fetch the next page. Use `react-intersection-observer` library for a simpler API.

## Q2: How do you implement debouncing in React?
**A:** Create a custom debounce hook: `function useDebounce(value, delay = 300) { const [debouncedValue, setDebouncedValue] = useState(value); useEffect(() => { const timer = setTimeout(() => setDebouncedValue(value), delay); return () => clearTimeout(timer); }, [value, delay]); return debouncedValue }`. Use in search: `const [query, setQuery] = useState(''); const debouncedQuery = useDebounce(query, 500); useEffect(() => { fetchResults(debouncedQuery) }, [debouncedQuery])`. This prevents excessive API calls on every keystroke by waiting until the user stops typing.

## Q3: How do you implement a virtualized list in React?
**A:** Use `react-window` for virtualizing large lists: `import { FixedSizeList as List } from 'react-window'; const Row = ({ index, style }) => (<div style={style}>Item {index}</div>); const VirtualList = ({ items }) => (<List height={400} itemCount={items.length} itemSize={50} width={300}>{Row}</List>)`. `react-window` only renders visible items (plus overscan), keeping DOM nodes minimal even for 100,000+ items. For variable heights, use `VariableSizeList`. For grid layouts, use `FixedSizeGrid`.

## Q4: How do you implement drag and drop in React?
**A:** Use the HTML5 Drag and Drop API with React state: `function DraggableList({ items }) { const [dragIndex, setDragIndex] = useState(null); const handleDragStart = (index) => setDragIndex(index); const handleDrop = (index) => { const newItems = [...items]; const [removed] = newItems.splice(dragIndex, 1); newItems.splice(index, 0, removed); setItems(newItems); setDragIndex(null) }; return items.map((item, i) => (<div key={item.id} draggable onDragStart={() => handleDragStart(i)} onDragOver={e => e.preventDefault()} onDrop={() => handleDrop(i)}>{item.name}</div>)) }`. For complex interactions, use libraries like `react-beautiful-dnd` (maintained) or `@dnd-kit/core` (modern).

## Q5: How do you implement a modal portal in React?
**A:** Use `createPortal` to render modals outside the component hierarchy: `function Modal({ isOpen, onClose, children }) { if (!isOpen) return null; return createPortal(<div className="overlay" onClick={onClose}><div className="modal" onClick={e => e.stopPropagation()}>{children}<button onClick={onClose}>Close</button></div></div>, document.getElementById('modal-root')) }`. Add `<div id="modal-root"></div>` to `index.html`. Portals ensure proper z-index stacking, avoid CSS overflow issues, and maintain React context propagation.

## Q6: How do you implement optimistic updates in React?
**A:** Update the UI immediately before the server confirms: `async function updateUser(id, data) { const previous = users; setUsers(prev => prev.map(u => u.id === id ? { ...u, ...data } : u)); try { await fetch(`/api/users/${id}`, { method: 'PUT', body: JSON.stringify(data) }); } catch { setUsers(previous); // Rollback on error showError('Failed to update') } }`. Libraries like TanStack Query provide built-in optimistic update support via `onMutate` callback: `useMutation({ mutationFn: updateUser, onMutate: async (newData) => { await queryClient.cancelQueries(['users']); const previous = queryClient.getQueryData(['users']); queryClient.setQueryData(['users'], old => old.map(...)); return { previous } }, onError: (err, newData, context) => { queryClient.setQueryData(['users'], context.previous) } })`.

## Q7: How do you implement component code splitting with React.lazy and Suspense?
**A:** Dynamically import components: `import React, { Suspense } from 'react'; const Dashboard = React.lazy(() => import('./Dashboard')); const Settings = React.lazy(() => import('./Settings')); function App() { return (<Suspense fallback={<PageLoader />}> <Routes> <Route path="/dashboard" element={<Dashboard />} /> <Route path="/settings" element={<Settings />} /> </Routes> </Suspense>) }`. For named exports, use `React.lazy(() => import('./Component').then(m => ({ default: m.NamedComponent })))`.

## Q8: How do you implement persistent state (localStorage) in React?
**A:** Create a custom hook that syncs state with localStorage: `function useLocalStorage(key, initialValue) { const [storedValue, setStoredValue] = useState(() => { try { const item = window.localStorage.getItem(key); return item ? ## Q10: How do you implement a custom hook for API data fetching with loading and error states?
**A:** Create a generic `useFetch` hook: `function useFetch(url) { const [state, setState] = useState({ data: null, isLoading: true, error: null }); useEffect(() => { let cancelled = false; setState(prev => ({ ...prev, isLoading: true })); fetch(url).then(r => { if (!r.ok) throw new Error(`HTTP ${r.status}`); return r.json() }).then(data => { if (!cancelled) setState({ data, isLoading: false, error: null }) }).catch(err => { if (!cancelled) setState({ data: null, isLoading: false, error: err.message }) }); return () => { cancelled = true } }, [url]); return state }`. The cleanup function prevents state updates on unmounted components. For production, use TanStack Query or SWR which handle caching, deduplication, and revalidation.

## Q12: How do you implement a countdown timer in React?
**A:** Use `useRef` for interval and `useEffect` for cleanup: `function Countdown({ seconds }) { const [timeLeft, setTimeLeft] = useState(seconds); const intervalRef = useRef(); useEffect(() => { intervalRef.current = setInterval(() => { setTimeLeft(prev => { if (prev <= 1) { clearInterval(intervalRef.current); return 0 } return prev - 1 }) }, 1000); return () => clearInterval(intervalRef.current) }, [seconds]); const minutes = Math.floor(timeLeft / 60); const secs = timeLeft % 60; return <div>{minutes}:{secs.toString().padStart(2, '0')}</div> }`. ## Q13: How do you implement undo/redo functionality in React?
**A:** Store a history of states: `function useUndo(initialState) { const [past, setPast] = useState([]); const [present, setPresent] = useState(initialState); const [future, setFuture] = useState([]); const setState = (newState) => { setPast(prev => [...prev, present]); setPresent(newState); setFuture([]) }; const undo = () => { if (past.length === 0) return; const previous = past[past.length - 1]; setPast(prev => prev.slice(0, -1)); setFuture(prev => [present, ...prev]); setPresent(previous) }; const redo = () => { if (future.length === 0) return; const next = future[0]; setFuture(prev => prev.slice(1)); setPast(prev => [...prev, present]); setPresent(next) }; const reset = (state) => { setPast([]); setPresent(state); setFuture([]) }; return { state: present, setState, undo, redo, reset, canUndo: past.length > 0, canRedo: future.length > 0 } }`. Implementations can also use immer for immutable updates.
