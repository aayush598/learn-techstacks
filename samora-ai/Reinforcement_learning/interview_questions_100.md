# Reinforcement Learning — 100 Interview Q&A
> Based on foundational RL theory, value-based methods, policy gradients, multi-agent RL, deep RL, and real-world deployment patterns.

---

## 1. Fundamentals & Core Concepts (Q1–Q20)

**Q1: What is Reinforcement Learning and how does it differ from supervised and unsupervised learning?**
A: RL is a paradigm where an agent learns to make decisions by interacting with an environment to maximize cumulative reward. Key differences: Supervised learning learns from labeled data (input→output mapping). Unsupervised learning finds patterns in unlabeled data. RL learns from trial-and-error with rewards/penalties — there is no correct answer, only feedback on actions taken. The agent must balance exploration (trying new actions) with exploitation (using known good actions).

**Q2: What are the key components of a reinforcement learning system?**
A: 1) Agent — the learner and decision-maker. 2) Environment — everything outside the agent, what it interacts with. 3) State (s) — current situation of the agent/environment. 4) Action (a) — what the agent can do. 5) Reward (r) — scalar feedback signal indicating how good the action was. 6) Policy (π) — the agent's strategy, mapping states to actions. 7) Value function — estimate of expected future rewards from a state/action. 8) Model (optional) — representation of environment dynamics.

**Q3: What is the Markov Decision Process (MDP) and why is it fundamental to RL?**
A: An MDP is a mathematical framework for modeling sequential decision-making. Defined by (S, A, P, R, γ): S = set of states, A = set of actions, P(s'|s,a) = transition probability, R(s,a,s') = reward function, γ = discount factor. The Markov property: the future depends only on the current state and action, not on the history. MDPs are fundamental because most RL problems are formulated as MDPs — the agent learns to maximize cumulative reward in an MDP.

**Q4: Explain the difference between the "reward" and "value" in RL.**
A: Reward is immediate — it's the feedback received right after taking an action. Value is the expected cumulative future reward from a state or state-action pair. A state can have low immediate reward but high value (e.g., in chess, sacrificing a piece leads to no immediate reward but high long-term value). The agent aims to maximize cumulative reward, which requires estimating values to make decisions that may sacrifice short-term reward for long-term gain.

**Q5: What is the "exploration vs exploitation" dilemma?**
A: The agent must choose between: Exploitation — take actions known to yield high reward based on current knowledge. Exploration — take new actions to gather information that may lead to better rewards in the future. Pure exploitation converges to a suboptimal solution (agent never discovers better options). Pure exploration never exploits what it learns. Balancing is critical. Common strategies: ε-greedy (random action with probability ε), Upper Confidence Bound (UCB), Thompson sampling.

**Q6: What is the "discount factor" γ and why is it used?**
A: The discount factor γ ∈ [0,1] determines how much the agent values future rewards vs immediate rewards. γ=0: agent only cares about immediate reward (myopic). γ≈1: agent values future rewards almost as much as immediate (far-sighted). Purposes: 1) Mathematical convenience — ensures infinite sums converge when γ<1. 2) Reflects uncertainty about the future. 3) Models preference for sooner rewards. 4) Reduces the effective horizon, making learning easier. Typical values: 0.9-0.99.

**Q7: What is the difference between "on-policy" and "off-policy" learning?**
A: On-policy: learns the value of the policy being followed (the behavior policy). The agent must follow the policy it's learning about. Example: SARSA. Off-policy: learns the value of a target policy while following a different behavior policy. This enables learning from data generated by any policy, including past data or human demonstrations. Example: Q-learning. Off-policy is more sample-efficient (reuses data) and supports learning from demonstration, but can be less stable.

**Q8: Explain the concept of "bootstrapping" in RL.**
A: Bootstrapping means updating value estimates using other value estimates (guesses based on guesses). For example, TD learning updates V(sₜ) using rₜ + γV(sₜ₊₁) — the current estimate of V(sₜ₊₁) is used to update V(sₜ). Bootstrapping introduces bias (estimates are inaccurate) but reduces variance and enables learning before episode completion. Monte Carlo methods don't bootstrap — they use complete returns — which is unbiased but higher variance and requires episode completion.

**Q9: What is the "credit assignment problem" in RL?**
A: The credit assignment problem is determining which actions in a sequence were responsible for the eventual reward. A reward received at the end of an episode may depend on actions taken many steps earlier. Should all previous actions be credited equally, or should some be blamed/rewarded more? TD learning addresses this through bootstrapping (credit flows backward through value estimates). Eligibility traces (λ-return) provide a spectrum between TD and Monte Carlo credit assignment.

**Q10: Define "policy," "value function," and "model" in the context of RL.**
A: 1) Policy (π) — the agent's behavior: a mapping from states to actions (π(s)→a) or a probability distribution (π(a|s)). 2) Value function: V(s) = expected discounted return from state s following π. Q(s,a) = expected discounted return from taking action a in state s, then following π. 3) Model — agent's representation of environment: transition model P(s'|s,a) and reward model R(s,a,s'). Methods: model-based (learn and use a model) vs model-free (learn directly from experience without a model).

**Q11: What is the "Bellman equation"?**
A: The Bellman equation expresses the value of a state in terms of the values of successor states. For the optimal value function: V*(s) = maxₐ Σ_{s'} P(s'|s,a)[R(s,a,s') + γV*(s')]. For Q: Q*(s,a) = Σ_{s'} P(s'|s,a)[R(s,a,s') + γ max_{a'} Q*(s',a')]. The Bellman equation is a recursive decomposition — the value of the current state/action equals the immediate reward plus the discounted value of the best next state/action. It forms the basis for all value-based RL algorithms.

**Q12: What is "dynamic programming" and how does it relate to RL?**
A: Dynamic programming (DP) refers to a collection of algorithms that compute optimal policies given a perfect model of the environment (transition probabilities and rewards). Policy iteration: evaluate a policy, improve it, repeat. Value iteration: iteratively apply the Bellman optimality operator. DP provides the theoretical foundation for RL — RL algorithms can be seen as DP with sampling (since they lack the model and must learn from data). Key difference: DP requires full knowledge of MDP; RL does not.

**Q13: Explain the "policy evaluation" and "policy improvement" steps.**
A: Policy evaluation: compute the value function V^π for a given policy π. Iteratively apply the Bellman expectation equation: V_{k+1}(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a)[R + γV_k(s')]. Policy improvement: given V^π, create a new policy π' that is greedy with respect to V^π: π'(s) = argmax_a Σ_{s'} P(s'|s,a)[R + γV^π(s')]. The policy improvement theorem guarantees π' >= π. Alternating evaluation and improvement converges to the optimal policy.

**Q14: What is the difference between "episodic" and "continuing" tasks?**
A: Episodic tasks have a clear terminal state (e.g., winning a game, reaching a goal). The agent-environment interaction breaks into episodes. Each episode ends in a terminal state, then resets. Continuing tasks have no terminal state — interaction goes on indefinitely (e.g., process control, stock trading). For continuing tasks, the discount factor γ<1 ensures the return is finite. Some algorithms (like Monte Carlo) require episodic tasks; others (like TD) work for both.

**Q15: What is "function approximation" and why is it necessary in RL?**
A: Function approximation represents the policy or value function using a parameterized function (e.g., neural network, linear model) instead of a table. Necessary because: 1) State/action spaces can be enormous (Go: 10^170 states). 2) Continuous state/action spaces have infinite states. 3) Generalization — learning in one state should generalize to similar states. 4) Compression — approximates the value/policy with fewer parameters. Deep RL uses neural networks as function approximators. Challenges: stability, divergence, catastrophic forgetting.

**Q16: What is "generalization" in RL and why is it challenging?**
A: Generalization means applying knowledge learned in one situation to similar but unseen situations. In RL, the agent must generalize across states, tasks, or environments. Challenges: 1) State aliasing — different states appear similar but require different actions. 2) Overfitting — policy performs well on training states but fails on test states. 3) Distribution shift — the state distribution changes as the policy improves. 4) Representation learning — the agent must learn features that support generalization. Techniques: function approximation, representation learning, domain randomization, data augmentation.

**Q17: What are the different types of RL agents based on what they learn?**
A: 1) Value-based — learn only a value function, derive policy implicitly (e.g., Q-learning, DQN). Policy is greedy with respect to Q. 2) Policy-based — learn the policy directly without a value function (e.g., REINFORCE, PPO). 3) Actor-Critic — learn both: actor (policy) and critic (value function). Actor uses critic's feedback to improve (e.g., A2C, SAC, TD3). 4) Model-based — learn a model of the environment (transition + reward), plan using the model. 5) Model-free — learn directly from experience without a model.

**Q18: Explain the "deadly triad" in deep RL.**
A: The deadly triad refers to the instability that arises when combining three elements: 1) Function approximation — neural networks for value/policy. 2) Bootstrapping — updating estimates using other estimates (TD learning). 3) Off-policy learning — learning about one policy while following another. Any two can be stable, but all three together can cause divergence. DQN addressed this with: experience replay (breaks correlations), target network (stabilizes bootstrapping targets), and careful reward/gradient clipping.

**Q19: What is "RLHF" (Reinforcement Learning from Human Feedback) and how does it work?**
A: RLHF uses human preferences to train RL agents, widely used for aligning LLMs. Process: 1) Collect human comparisons of model outputs (which response is better). 2) Train a reward model that predicts human preferences. 3) Use the reward model as the reward signal for RL training (typically PPO). The RL agent (the LLM) learns to generate outputs that maximize the reward model's score. RLHF enables training on subjective, hard-to-specify objectives (helpfulness, harmlessness).

**Q20: What is "multi-objective reinforcement learning"?**
A: In multi-objective RL, the agent must optimize multiple conflicting objectives (e.g., speed vs safety, profit vs risk). Instead of a single reward scalar, there is a reward vector. Approaches: 1) Scalarization — weighted sum combines objectives into a single reward (simple but requires weight tuning). 2) Pareto optimality — find policies where no objective can be improved without degrading another. 3) Conditioned policies — condition the policy on the desired objective weights. 4) Multi-policy approaches — learn multiple policies for different trade-offs.

## 2. Value-Based Methods (Q21–Q35)

**Q21: Explain "Temporal Difference (TD) Learning".**
A: TD learning combines ideas from Monte Carlo (learning from experience) and dynamic programming (bootstrapping). The TD update: V(sₜ) ← V(sₜ) + α[rₜ + γV(sₜ₊₁) - V(sₜ)]. The TD error δ = rₜ + γV(sₜ₊₁) - V(sₜ) is the difference between the predicted value and the better estimate using the actual reward and next state's value. TD(0) is the simplest form — uses one-step lookahead. Advantages: learns from incomplete episodes, online/incremental, lower variance than MC.

**Q22: What is "Q-learning" and why is it considered off-policy?**
A: Q-learning learns the optimal Q-function directly. Update rule: Q(s,a) ← Q(s,a) + α[r + γ max_{a'} Q(s',a') - Q(s,a)]. It's off-policy because it uses the max over next actions (which corresponds to the greedy policy) regardless of the actual action taken (behavior policy). The agent can follow an exploratory policy (e.g., ε-greedy) while learning the optimal Q-function. Q-learning converges to Q* under mild conditions (all state-action pairs visited infinitely often, appropriate learning rate schedule).

**Q23: Explain "SARSA" (State-Action-Reward-State-Action) and how it differs from Q-learning.**
A: SARSA update: Q(s,a) ← Q(s,a) + α[r + γQ(s',a') - Q(s,a)] where a' is the action actually taken in the next state (following the behavior policy). SARSA is on-policy — it learns Q^π for the policy being followed. Key difference: Q-learning uses max_{a'} Q(s',a') (optimistic, assumes best action), SARSA uses Q(s',a') (realistic, uses actual action). This means: Q-learning learns optimal Q* regardless of exploration, while SARSA accounts for exploration risk. SARSA tends to learn safer policies in risky environments.

**Q24: What is "Deep Q-Network" (DQN) and what innovations made it work?**
A: DQN (Mnih et al., 2015) uses a neural network to approximate Q(s,a) for high-dimensional state spaces (Atari games from pixels). Key innovations: 1) Experience replay — stores transitions (s,a,r,s') in a buffer, samples random batches for training. Breaks correlations between consecutive samples, improves data efficiency. 2) Target network — a separate Q-network with frozen parameters is used to compute TD targets. Periodically updated from the online network. Reduces moving-target problem and stabilizes learning. 3) Reward clipping — clips rewards to [-1,1] for stability.

**Q25: What is "Double Q-learning" and why is it needed?**
A: Standard Q-learning uses the same Q-values for both action selection (max_a) and action evaluation (computing the target). This causes maximization bias — overestimation of Q-values because noise in estimates makes max over actions a biased estimate of the maximum. Double Q-learning decouples selection and evaluation using two separate Q-functions: one for selecting the best action, the other for estimating its value. In deep RL, Double DQN (DDQN) uses the online network for selection and the target network for evaluation.

**Q26: Explain "Dueling DQN" architecture.**
A: Dueling DQN decomposes Q(s,a) into two streams: V(s) — state-value function (how good is this state), and A(s,a) — advantage function (how much better is this action compared to others). Q(s,a) = V(s) + (A(s,a) - mean_a A(s,a)). The advantage stream has zero mean to ensure identifiability. Benefits: the network can learn which states are valuable without having to learn the effect of each action in every state. More efficient learning, especially when actions don't affect the outcome equally.

**Q27: What is "Prioritized Experience Replay"?**
A: Standard experience replay samples transitions uniformly. Prioritized replay samples more frequently those transitions from which the agent can learn more — measured by TD error magnitude |δ|. Transitions with high TD error are "surprising" and more informative. Sampling probability proportional to priority: P(i) = p_i^α / Σ p_j^α. New transitions get maximum priority. Importance sampling weights correct the bias introduced by non-uniform sampling. Results in faster learning, especially in early stages.

**Q28: Explain "Rainbow DQN".**
A: Rainbow combines six DQN improvements into a single agent: 1) DQN — base architecture. 2) Double DQN — reduces overestimation. 3) Prioritized replay — focuses on informative transitions. 4) Dueling networks — separate V and A streams. 5) Multi-step learning — uses n-step returns. 6) Distributional RL — learns the full distribution of returns, not just the expectation. 7) Noisy nets — replaces ε-greedy exploration with learnable noise in network weights. Rainbow achieved state-of-the-art on Atari at the time.

**Q29: What is "Distributional RL" and how does C51 (Categorical DQN) work?**
A: Instead of learning the expected Q-value, distributional RL learns the full probability distribution of returns. C51 represents the return distribution as a categorical distribution over 51 fixed support atoms. The distribution is projected onto the support atoms. Benefits: 1) More expressive — captures risk, uncertainty, multimodality. 2) More stable learning — distributional Bellman updates preserve distribution structure. 3) Better performance empirically — C51 significantly outperformed DQN on Atari.

**Q30: What is "n-step TD learning"?**
A: n-step TD learning looks ahead n steps before bootstrapping. The target is the n-step return: G_{t:t+n} = r_t + γr_{t+1} + ... + γ^{n-1}r_{t+n-1} + γ^n V(s_{t+n}). It is a spectrum between TD(0) (n=1, high bias, low variance) and Monte Carlo (n=∞, unbiased, high variance). n-step TD often learns faster than TD(0) because it propagates rewards more quickly. In deep RL, n-step returns are used in A3C, Rainbow, and many actor-critic methods.

**Q31: What is "eligibility traces" and how does TD(λ) work?**
A: Eligibility traces bridge the gap between TD(0) and Monte Carlo. TD(λ) uses λ ∈ [0,1] to weight different n-step returns. The λ-return: G^λ_t = (1-λ) Σ_{n=1}^∞ λ^{n-1} G_{t:t+n}. λ=0 is TD(0), λ=1 is MC. Eligibility traces maintain an "eligibility" e(s) for each state, decaying by γλ each step. When a TD error occurs, all recently visited states are updated proportional to their eligibility. Provides smooth credit assignment across time steps.

**Q32: How does Q-learning handle continuous state spaces?**
A: For continuous states, Q-learning requires function approximation since tabular representation is impossible. Approaches: 1) Discretization — divide continuous dimensions into bins, use tabular Q-learning (loses precision, curse of dimensionality). 2) Linear function approximation — Q(s,a) = w^T φ(s,a) where φ are handcrafted features. 3) Neural networks — DQN uses a neural network to approximate Q(s,a) directly from raw state inputs. 4) Tile coding — overlapping grids that provide coarse coding of continuous space.

**Q33: What is "Fitted Q-Iteration" (FQI)?**
A: FQI is a batch (offline) RL algorithm that uses a supervised learning approach. At each iteration: 1) Use current Q-function to compute target values for all transitions in a dataset: y_i = r_i + γ max_{a'} Q(s'_i, a'). 2) Train a new Q-function (e.g., neural network, random forest) to predict targets from (s_i, a_i). 3) Repeat. FQI is data-efficient (reuses data), supports any function approximator, and is stable (supervised learning at each step). Limitation: computationally expensive for many iterations.

**Q34: Explain the "greedy in the limit with infinite exploration" (GLIE) condition.**
A: GLIE is a condition for convergence of on-policy RL algorithms. It requires: 1) All state-action pairs are visited infinitely often (infinite exploration). 2) The policy converges to a greedy policy (limit greedy). Example: ε-greedy with ε → 0 as t → ∞ (e.g., ε = 1/t). Under GLIE, SARSA converges to the optimal policy. Q-learning does not require GLIE (it's off-policy) but still benefits from sufficient exploration.

**Q35: What are the convergence guarantees for Q-learning?**
A: Tabular Q-learning converges to Q* with probability 1 under: 1) All state-action pairs visited infinitely often. 2) Learning rates satisfy Robbins-Monro conditions: Σ α_t = ∞, Σ α_t² < ∞ (e.g., α_t = 1/t). 3) The MDP is finite and stationary. For Q-learning with function approximation (deep Q-learning), there are no general convergence guarantees — the deadly triad can cause divergence. Empirical stability is achieved through techniques like target networks, replay buffers, and gradient clipping.

## 3. Policy-Based & Actor-Critic Methods (Q36–Q50)

**Q36: What is the "policy gradient theorem"?**
A: The policy gradient theorem provides an analytic expression for the gradient of the expected return J(θ) with respect to policy parameters θ: ∇J(θ) = E_{τ∼π_θ}[Σ_t ∇_θ log π_θ(a_t|s_t) * G_t]. Where G_t is the return from time t. This gradient tells us how to adjust policy parameters to increase expected return without requiring differentiation of the state distribution (which depends on the policy). The key insight: ∇_θ log π_θ(a|s) * G_t is an unbiased estimate of the gradient.

**Q37: Explain the REINFORCE algorithm.**
A: REINFORCE (Williams, 1992) is a Monte Carlo policy gradient method. Algorithm: 1) Generate an episode using current policy π_θ. 2) For each step t, compute return G_t (cumulative discounted reward from t). 3) Update: θ ← θ + α * G_t * ∇_θ log π_θ(a_t|s_t). REINFORCE is simple, unbiased, and model-free. Drawbacks: high variance (uses full episode returns), requires episodic tasks, slow convergence. Reducing variance: subtract a baseline (e.g., running average of returns) to get θ ← θ + α * (G_t - b) * ∇_θ log π_θ(a_t|s_t).

**Q38: How does the "baseline" reduce variance in policy gradients?**
A: A baseline b(s) is subtracted from the return G_t without introducing bias: ∇J = E[(G_t - b(s_t)) ∇_θ log π_θ(a_t|s_t)]. The baseline can be any function of the state (but not the action) — common choices: 1) Constant baseline — average return across all episodes. 2) State-dependent baseline — the value function V(s) (best choice). Intuition: the term (G_t - V(s_t)) represents the "advantage" — how much better this action was compared to average. Good actions get positive advantage, bad actions get negative. This dramatically reduces variance.

**Q39: What is the "Actor-Critic" architecture?**
A: Actor-Critic combines policy-based and value-based methods. Two components: 1) Actor (policy) — π_θ(a|s), learns the policy using policy gradient with feedback from the critic. 2) Critic (value function) — V_φ(s) or Q_φ(s,a), learns to estimate the value function to provide a lower-variance baseline for the actor. The actor updates: θ ← θ + α * (r + γV(s') - V(s)) * ∇_θ log π(a|s). The critic updates: minimize TD error. Advantages: lower variance than REINFORCE, supports online/incremental learning, continuous actions.

**Q40: Explain "Advantage Actor-Critic" (A2C / A3C).**
A: A2C uses the advantage function A(s,a) = Q(s,a) - V(s) in the policy gradient: ∇J = E[∇_θ log π(a|s) * A(s,a)]. The advantage is estimated using n-step returns: A_t = G_{t:t+n} - V(s_t). A3C (Asynchronous Advantage Actor-Critic) runs multiple workers in parallel, each interacting with its own environment copy. Workers asynchronously update a shared model. A2C is the synchronous version — workers complete their steps, then update together. Benefits: parallel exploration, decorrelated updates.

**Q41: What is "Proximal Policy Optimization" (PPO)?**
A: PPO (Schulman et al., 2017) is a policy gradient method that improves stability by constraining policy updates. Key idea: take the biggest possible update that does not destabilize the policy. PPO-clip modifies the surrogate objective: L^{CLIP}(θ) = E[min(r_t(θ) A_t, clip(r_t(θ), 1-ε, 1+ε) A_t)], where r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t) is the probability ratio. The clip prevents r_t from moving outside [1-ε, 1+ε]. PPO is simpler to implement, more robust, and more sample-efficient than TRPO.

**Q42: How does PPO differ from TRPO (Trust Region Policy Optimization)?**
A: Both constrain policy updates to a trust region. TRPO uses a hard constraint: E[KL(π_old || π_θ)] ≤ δ, solved via conjugate gradient. The constraint ensures updates stay within a region where the policy approximation is accurate. TRPO is theoretically sound but complex. PPO approximates the same idea with a simple clipped objective — no conjugate gradient, no KL computation. PPO is easier to implement, computationally lighter, and in practice often outperforms TRPO. PPO is the default policy gradient algorithm in most RL libraries.

**Q43: Explain "Soft Actor-Critic" (SAC).**
A: SAC (Haarnoja et al., 2018) is an off-policy actor-critic method that maximizes both expected return and entropy (randomness). Objective: J(π) = Σ_t E[γ^t (r(s_t,a_t) + α H(π(·|s_t)))]. The entropy term H(π(·|s)) encourages exploration and prevents premature convergence. SAC uses: 1) Twin Q-networks (like TD3) to reduce overestimation. 2) A learnable temperature α that adjusts entropy regularization. 3) Off-policy updates with experience replay. SAC is state-of-the-art for continuous control, known for sample efficiency and stability.

**Q44: What is "Twin Delayed DDPG" (TD3)?**
A: TD3 (Fujimoto et al., 2018) addresses overestimation bias in actor-critic methods. Three key improvements over DDPG: 1) Clipped double Q-learning — two Q-networks, use the minimum Q-value for target computation (reduces overestimation). 2) Delayed policy updates — update the actor less frequently than the critic (every d steps). 3) Target policy smoothing — add noise to target actions (regularizes Q-function, prevents sharp peaks). TD3 is robust and sample-efficient for continuous action spaces.

**Q45: Explain the difference between on-policy and off-policy actor-critic methods.**
A: On-policy actor-critic (A2C, PPO): use data generated by the current policy. Each batch of data is used once (for one gradient update), then discarded. More stable but less sample-efficient. Off-policy actor-critic (SAC, TD3, DDPG): use experience replay — data from old policies can be reused. More sample-efficient but less stable (deadly triad). Off-policy methods typically use techniques like target networks, clipped double Q-learning, and delayed updates to maintain stability.

**Q46: What is "Deep Deterministic Policy Gradient" (DDPG)?**
A: DDPG (Lillicrap et al., 2016) extends DQN to continuous action spaces using an actor-critic architecture with: 1) An actor network that outputs deterministic actions (μ(s)). 2) A critic network that evaluates Q(s, μ(s)). 3) Experience replay for sample efficiency. 4) Target networks for both actor and critic (soft-updated: θ_target ← τθ + (1-τ)θ_target). DDPG uses exploration by adding noise to actions (Ornstein-Uhlenbeck process). DDPG is sensitive to hyperparameters and can be unstable — TD3 and SAC are preferred alternatives.

**Q47: What is "Natural Policy Gradient" and how does it differ from vanilla policy gradient?**
A: Vanilla policy gradient uses the Euclidean gradient which does not account for how the policy distribution changes. Natural policy gradient uses the natural gradient — it follows the steepest ascent in the policy space rather than parameter space. It preconditions the gradient with the inverse Fisher Information Matrix: ∇_natural J = F(θ)^{-1} ∇_θ J(θ). The Fisher matrix captures the sensitivity of the policy distribution to parameter changes. Natural gradient is invariant to parameterization and provides more stable, efficient updates. TRPO approximates natural gradient with a KL constraint.

**Q48: Explain "Generalized Advantage Estimation" (GAE).**
A: GAE (Schulman et al., 2016) provides a flexible, low-variance estimate of the advantage function. GAE(λ, γ) = Σ_{l=0}^∞ (γλ)^l δ_{t+l}, where δ_t = r_t + γV(s_{t+1}) - V(s_t) is the TD error. GAE interpolates between: λ=0 (one-step TD advantage — high bias, low variance) and λ=1 (Monte Carlo advantage — unbiased, high variance). GAE(0.95) is a common choice. GAE significantly reduces variance in policy gradient updates and is used in PPO, A2C, and many modern RL algorithms.

**Q49: What is the "reparameterization trick" and how is it used in policy gradients?**
A: The reparameterization trick expresses a random variable as a deterministic function of a base random variable and parameters: a = μ_θ(s) + σ_θ(s) * ε, where ε ∼ N(0,1). This allows gradients to flow through the sampling process. Used in: 1) SAC — reparameterized sampling for policy loss. 2) Continuous policy gradients — enables backpropagation through action sampling. 3) Variational inference in model-based RL. Benefits: lower variance gradient estimates compared to the score function (REINFORCE) gradient.

**Q50: What is "Maximum a Posteriori Policy Optimization" (MPO)?**
A: MPO (Abdolmaleki et al., 2018) is a policy optimization method that formulates policy improvement as an expectation-maximization (EM) problem. E-step: compute a improved non-parametric policy distribution by weighting actions by their Q-values (using exponentiated Q-values). M-step: project this improved distribution back onto the parametric policy space by minimizing KL divergence. MPO is robust, sample-efficient, and works well for both discrete and continuous control. It avoids the tuning challenges of PPO's clipping hyperparameter.

## 4. Model-Based RL & Planning (Q51–Q60)

**Q51: What is "model-based reinforcement learning"?**
A: Model-based RL learns a model of the environment (transition dynamics P(s'|s,a) and reward function R(s,a)) and uses it for planning or policy optimization. The agent: 1) Interacts with the real environment to collect data. 2) Learns a model from the data (supervised learning). 3) Uses the model to simulate experience or plan actions. Advantages: sample efficiency (model generates synthetic experience), planning capability (look ahead), transferability (model can be reused). Disadvantages: model bias (errors compound), computational cost of planning.

**Q52: What is "Dyna-style" architecture?**
A: The Dyna architecture (Sutton, 1991) integrates learning, planning, and acting. The agent: 1) Interacts with real environment (learn from real experience). 2) Updates the model (learn from real experience). 3) Plans using the model — generates simulated experience from the model and uses it for learning (same RL algorithm as real experience). Dyna-Q: combines Q-learning with model-based planning. At each step: perform one real Q-update, then N simulated Q-updates using the model. Dyna is sample-efficient because model simulations augment real experience.

**Q53: Explain "Monte Carlo Tree Search" (MCTS).**
A: MCTS is a planning algorithm that builds a search tree through random simulation. Four steps per iteration: 1) Selection — traverse tree from root using UCB (Upper Confidence Bounds) to select most promising nodes. 2) Expansion — add a new child node to the tree. 3) Rollout (simulation) — play random or light-policy actions from the new node to termination. 4) Backpropagation — update win counts and visit counts up the tree. After many iterations, the best action at the root is chosen. MCTS was key to AlphaGo's success.

**Q54: How does "AlphaGo" and "AlphaZero" use RL?**
A: AlphaGo combines supervised learning (from human games), RL (self-play), and MCTS. Components: 1) Policy network — predicts the best move. 2) Value network — predicts the winner from a position. 3) MCTS — uses policy network for guided search, value network for leaf evaluation. AlphaZero simplifies: no human data, learns entirely from self-play using a single neural network that outputs both policy (move probabilities) and value (win probability). MCTS guides self-play, and search results train the network. This achieved superhuman performance in Go, Chess, and Shogi.

**Q55: What is "World Models" approach?**
A: World Models (Ha & Schmidhuber, 2018) learn a compressed representation of the environment. Three components: 1) Vision (V) model — autoencoder that compresses high-dimensional observations (images) into a latent representation. 2) Memory (M) model — RNN that predicts future latent states given actions. 3) Controller (C) model — simple policy that acts in the latent space. The agent trains the controller entirely inside the learned world model (no real environment interaction needed after model is trained). Demonstrates that agents can "dream" to learn policies.

**Q56: Explain "Model Predictive Control" (MPC) in RL.**
A: MPC is a planning-based control method: 1) At each timestep, use the learned model to simulate possible action sequences. 2) Evaluate each sequence using a reward/cost function. 3) Select the first action of the best sequence. 4) Execute it, observe the outcome, and replan at the next timestep. MPC provides robust control because it continuously replans. In RL, MPC is used as the policy (planning replaces explicit policy learning). Random Shooting (CEM, cross-entropy method) is a common MPC variant that samples action sequences and selects the best.

**Q57: What is "Dreamer" (DreamerV1/V2/V3)?**
A: Dreamer (Hafner et al., 2020) is a model-based RL agent that learns a world model from experience and uses it to train a policy entirely within "dreams" (latent imagination). Components: 1) World model learns to predict future latent states (RSSM — Recurrent State Space Model). 2) Actor-critic trained on imagined trajectories from the world model. Dreamer learns behaviors by propagating value gradients through the learned world model. Successfully learned complex tasks from pixels (Atari, DM Control, Minecraft). DreamerV2 improved with categorical latents; DreamerV3 introduced scaling improvements.

**Q58: What is the difference between "shooting" and "dynamic programming" approaches to planning?**
A: Shooting methods (MCTS, random shooting, CEM): simulate multiple action sequences from the current state, evaluate them, and pick the best. Simple, parallelizable, works with non-differentiable models. Dynamic programming approaches (value iteration, policy iteration): compute optimal value function for all states, then derive policy. Requires manageable state space or function approximation. Shooting is myopic and recomputed each step; DP provides a full policy. In deep RL: shooting is used for short-horizon planning; DP is used for long-horizon value learning.

**Q59: What is "data augmentation" in model-based RL?**
A: Model-generated data augments real experience to improve sample efficiency. Approaches: 1) Hallucinated trajectories — run the policy through the model to generate synthetic episodes. 2) Model-based value expansion (MVE) — use the model to extend n-step returns further into the future. 3) Branching — from a real state, simulate multiple action branches through the model. 4) Priority sweeping — prioritize updates to states whose values changed significantly. The key challenge: model errors compound, so synthetic data quality degrades with trajectory length.

**Q60: What are the main failure modes of model-based RL?**
A: 1) Model bias — the learned model is inaccurate, leading to policies that exploit model errors. 2) Compounding errors — small prediction errors accumulate over long trajectories, making multi-step predictions unreliable. 3) Distribution shift — the policy visits states not well-represented in the training data, where model predictions are poor. 4) Computational cost — planning/optimization with complex models (e.g., neural networks) is expensive. 5) Model uncertainty — agent fails to account for regions where the model is uncertain. Mitigations: ensemble models, uncertainty-aware planning, short-horizon planning, and Dyna-style partial planning.

## 5. Exploration & Reward Design (Q61–Q75)

**Q61: Explain "ε-greedy" exploration and its limitations.**
A: ε-greedy: with probability ε, choose a random action; with probability 1-ε, choose the greedy action. Simple and widely used. Limitations: 1) Explores uniformly — wastes time on obviously bad actions. 2) No state-dependence — explores the same rate in all states regardless of uncertainty. 3) No memory — does not track which actions have been tried. 4) Linear regret — regret grows linearly with time because exploration never decreases to zero (unless ε is annealed). Despite limitations, ε-greedy is effective in practice, especially with annealing.

**Q62: What is "Upper Confidence Bound" (UCB) exploration?**
A: UCB selects actions based on optimistic estimates: a_t = argmax_a [Q(a) + c * sqrt(ln(t) / N(a))], where N(a) is the visit count of action a. The second term is the confidence bound — actions with fewer visits have higher uncertainty and are preferred. UCB balances exploration and exploitation naturally: uncertain actions are explored; as uncertainty decreases, exploitation takes over. UCB achieves logarithmic regret (much better than ε-greedy's linear regret). The UCB formula is derived from Hoeffding's inequality (optimism in the face of uncertainty).

**Q63: What is "Thompson Sampling"?**
A: Thompson Sampling maintains a Bayesian posterior distribution over Q-values: P(Q(a) | data). At each step, sample from the posterior and act greedily with respect to the sample. Actions are selected with probability proportional to being optimal. Thompson Sampling: 1) Handles uncertainty naturally. 2) Achieves optimal regret asymptotically (Bayesian regret bounds). 3) Works for complex models (Gaussian processes, Bayesian neural networks). 4) Can incorporate prior knowledge. Limitation: maintaining a full posterior is computationally expensive.

**Q64: Explain "count-based exploration" in RL.**
A: Count-based exploration gives exploration bonuses to states or actions that have been visited fewer times. Bonus: r' = r + β / sqrt(N(s)) or r' = r + β / N(s). Encourages the agent to visit novel states. In tabular settings, simple visit counts work. In deep RL (high-dimensional states), counting is challenging — solutions: 1) Pseudo-counts — train a density model (e.g., PixelCNN) over states; count is derived from prediction gain. 2) Hash-based — hash states into discrete buckets and count bucket visits. 3) Exemplar models — classify whether a state is novel vs seen.

**Q65: What is "curiosity-driven exploration"?**
A: Curiosity-driven exploration provides intrinsic rewards based on prediction error or information gain. Approaches: 1) ICM (Intrinsic Curiosity Module, Pathak et al., 2017) — trains a forward dynamics model to predict next state features given action; prediction error is the intrinsic reward. Novel states with unpredictable dynamics give high reward. 2) Disagreement ensemble — train an ensemble of models; variance (disagreement) measures epistemic uncertainty and provides intrinsic reward. 3) Information gain — reward for actions that reduce model uncertainty. Curiosity helps exploration in sparse-reward environments.

**Q66: What is "Random Network Distillation" (RND)?**
A: RND (Burda et al., 2018) is a curiosity method for hard-exploration problems. Two networks: 1) A fixed, randomly initialized target network f* that maps observations to embeddings. 2) A predictor network f_θ trained to match f* outputs on visited states. The prediction error ||f_θ(s) - f*(s)||² is the intrinsic reward. Novel states have high prediction error (predictor hasn't been trained on them). RND addresses the "noisy TV problem" — prevents the agent from getting stuck on stochastic elements (since RND errors persist for unreducible noise). RND was key to solving Montezuma's Revenge.

**Q67: Explain "Go-Explore" exploration algorithm.**
A: Go-Explore (Ecoffet et al., 2021) is a two-phase exploration method that solved previously unsolved Atari games. Phase 1 — Explore: 1) Remember all visited states (archive). 2) Return to a promising state from archive. 3) Take a random action. 4) Add the resulting state to archive. Phase 2 — Robustify: train a policy to reach high-reward states. Key insight: traditional exploration methods explore from the current state, which may require backtracking through hard-to-reach states. Go-Explore explicitly returns to promising states before exploring, avoiding the "detachment" problem.

**Q68: What are the challenges in sparse reward environments?**
A: Sparse rewards provide feedback only when the agent reaches a goal state, making learning extremely difficult. Challenges: 1) Random exploration rarely reaches the goal — probability of success is exponentially small in horizon length. 2) No gradient information — no reward signal to guide policy improvement between successes. 3) Credit assignment — difficult to identify which actions led to the rare success. Solutions: 1) Reward shaping — provide intermediate rewards (requires domain knowledge). 2) Curiosity/exploration bonuses — intrinsic rewards for novelty. 3) Hindsight experience replay — treat failed trajectories as successes with relabeled goals. 4) Hierarchical RL — learn sub-goals.

**Q69: What is "Reward Shaping" and what is the "potential-based" approach?**
A: Reward shaping adds additional rewards to the environment's reward to guide learning: R' = R + F. Potential-based shaping (Ng, Harada, Russell, 1999) ensures the optimal policy is preserved: F(s, a, s') = γΦ(s') - Φ(s), where Φ is a potential function (any function of state). The insight: any potential-based shaping is equivalent to initializing the value function to Φ, not changing the optimal policy. Non-potential shaping can change the optimal policy and cause suboptimal behavior. Example: Φ(s) = negative distance to goal.

**Q70: Explain "Hindsight Experience Replay" (HER).**
A: HER (Andrychowicz et al., 2017) addresses sparse reward problems in goal-conditioned RL. Key insight: even when the agent fails to achieve the intended goal, it achieves some other outcome. HER replays failed trajectories with a relabeled goal — the goal is replaced by the state actually achieved. The agent learns: "if I wanted to reach the state I actually reached, those actions would have been optimal." This provides a learning signal from every episode. HER dramatically improves sample efficiency in goal-reaching tasks.

**Q71: What is "inverse reinforcement learning" (IRL)?**
A: IRL infers the reward function from expert demonstrations. Instead of learning a policy from rewards (forward RL), IRL learns what reward function the expert is optimizing. Approaches: 1) Maximum margin IRL — find reward that makes expert actions have higher value than alternatives. 2) Maximum entropy IRL (Ziebart) — find reward that maximizes entropy of demonstrated trajectories while matching feature expectations. 3) Adversarial IRL (GAIL — Generative Adversarial Imitation Learning) — discriminator distinguishes expert from agent, agent tries to fool it. IRL is used when specifying rewards is difficult but demonstrations are available.

**Q72: Explain "Intrinsic Motivation" in RL.**
A: Intrinsic motivation drives exploration through internally-generated rewards rather than external task rewards. Types: 1) Knowledge-based — reward for reducing uncertainty (prediction error, information gain). 2) Competence-based — reward for mastering skills (empowerment, learning progress). 3) Novelty-based — reward for visiting novel states. Intrinsic motivation enables: learning without external rewards, skill discovery, and transfer learning. The challenge: balancing intrinsic and extrinsic rewards so the agent doesn't get distracted by intrinsic exploration.

**Q73: What is "Empowerment" as an intrinsic motivation?**
A: Empowerment measures how much influence an agent has over its future states — the channel capacity between an agent's actions and future states. A state with high empowerment has many actions that lead to distinguishable future states. Empowerment as intrinsic reward: the agent seeks states where it has more control. Benefits: 1) Task-agnostic — does not require task rewards. 2) Produces diverse, controllable behaviors. 3) Naturally leads to skill discovery. Limitation: computing empowerment exactly is intractable for high-dimensional spaces.

**Q74: What are "Reward Models" in the context of learned rewards?**
A: Reward models approximate the true reward function when: 1) The true reward is expensive to compute (simulation-based). 2) The reward is based on human judgment (RLHF). 3) The reward is unknown and must be inferred (IRL). Approaches: 1) Learn a parametric reward model from human feedback (RLHF). 2) Learn from demonstrations (inverse RL). 3) Learn from pairwise comparisons. 4) Learn reward from multiple sources. Challenges: reward hacking (agent exploits reward model), distribution shift (reward model is inaccurate for states not seen during training).

**Q75: What is "reward hacking" and how is it mitigated?**
A: Reward hacking occurs when the agent finds ways to maximize the reward signal without actually achieving the intended goal. Examples: 1) Boat racing agent learns to circle and collect rewards instead of finishing. 2) Cleaning robot learns to tip over the trash can and watch debris fall out repeatedly. 3) RLHF model learns to exploit the reward model. Mitigations: 1) Careful reward design — avoid giving rewards for easily exploitable behaviors. 2) Reward verification — human oversight of agent behavior. 3) Adversarial reward learning — train reward model to be robust. 4) Multiple reward sources — diversify reward signal. 5) Constrained RL — penalize undesirable behaviors directly.

## 6. Multi-Agent & Advanced RL Topics (Q76–Q90)

**Q76: What is "multi-agent reinforcement learning" (MARL) and what makes it challenging?**
A: MARL extends RL to environments with multiple agents that learn simultaneously. Unique challenges: 1) Non-stationarity — other agents' policies change, making the environment non-stationary from any agent's perspective. 2) Scalability — joint action space grows exponentially. 3) Credit assignment — which agent contributed to the team reward? 4) Communication — agents may need to learn to communicate. 5) Equilibrium selection — many possible Nash equilibria, agents must coordinate on one. 6) Partial observability — each agent sees only part of the global state.

**Q77: Explain "independent Q-learning" (IQL) and its limitations in MARL.**
A: IQL: each agent runs standard Q-learning independently, treating other agents as part of the environment. Simple and scalable. Limitations: 1) Non-stationarity — other agents' policies change, violating the Markov property. Past Q-updates become invalid. 2) Exploration conflict — one agent's exploration changes the environment for others. 3) No coordination — agents may learn to avoid each other but not to cooperate. Despite these issues, IQL works in practice for some domains because: learning rates are slow enough that non-stationarity is manageable, and noise can help exploration.

**Q78: What is "Centralized Training with Decentralized Execution" (CTDE)?**
A: CTDE is the dominant paradigm in MARL. During training, agents have access to global information (all agents' states, actions, rewards). During execution, each agent acts using only its local observations. This gives the best of both worlds: stable training with full information, and practical decentralized deployment. Key CTDE algorithms: 1) MADDPG — centralized critic with per-agent actors. 2) QMIX/VDN — centralized Q-function mixing. 3) MAPPO — centralized value function with decentralized policies. CTDE addresses non-stationarity during training (agents see other agents' policies).

**Q79: What is "MADDPG" (Multi-Agent Deep Deterministic Policy Gradient)?**
A: MADDPP (Lowe et al., 2017) extends DDPG to multi-agent with CTDE. Each agent has: actor (local observations → action) and centralized critic (all agents' observations and actions → Q-value for that agent). The critic has access to all agents' information during training. Key benefits: 1) Handles cooperative, competitive, and mixed environments. 2) Centralized critic stabilizes training. 3) Agents can learn complex coordination. Limitations: critics scale poorly (input dimension grows with number of agents), requires knowing all agents during training.

**Q80: Explain "QMIX" and "VDN" for cooperative MARL.**
A: Both decompose the joint Q-function into per-agent Q-functions for scalability. VDN (Value Decomposition Networks): Q_tot(s, a) = Σ_i Q_i(s, a_i). Simple sum decomposition. QMIX: Q_tot is a monotonic mixing of per-agent Q-values using a hypernetwork conditioned on the global state. QMIX is more expressive than VDN while ensuring ∂Q_tot/∂Q_i ≥ 0 (Individual-Global-Max property). This property ensures that greedily selecting actions per-agent (argmax Q_i) yields the globally optimal joint action. QMIX is widely used in cooperative MARL benchmarks (SMAC, Starcraft).

**Q81: What is "Mean Field MARL"?**
A: Mean Field MARL simplifies MARL by approximating the interaction between an agent and all other agents as an interaction with the "mean" or average effect of the population. The Q-function for agent i depends on the agent's state/action and the mean action of neighboring agents. This reduces the complexity from exponential (in number of agents) to linear. Mean field approximation is valid for large populations where individual agent differences average out. Used in: large-scale multi-agent systems, traffic control, crowd simulation. Limitations: less accurate for small numbers of agents or significant individual differences.

**Q82: What is "hierarchical reinforcement learning" (HRL)?**
A: HRL decomposes complex tasks into a hierarchy of sub-tasks, each managed by its own policy. Top-level policy selects sub-goals; lower-level policies achieve sub-goals. Key frameworks: 1) Options framework (Sutton et al.) — temporally extended actions (options) with initiation sets and termination conditions. 2) FeUdal Networks (FuN) — manager selects goals, worker executes actions to achieve goals. 3) HIRO — off-policy HRL with goal relabeling. HRL addresses: long-horizon tasks (credit assignment over many steps), exploration (explore at sub-goal level), transfer learning (reuse sub-policies).

**Q83: Explain the "Options" framework in HRL.**
A: An option (Sutton, Precup, Singh, 1999) is a temporally extended action consisting of: 1) Initiation set I ⊆ S — states where the option can be initiated. 2) Policy π_ω: S × A → [0,1] — the option's internal policy. 3) Termination condition β_ω: S → [0,1] — probability of terminating in each state. Options enable hierarchical planning: the agent selects among options (high-level), and the option's policy takes over (low-level). Options can be learned or predefined. The options framework provides a formal foundation for HRL with the SMDP (Semi-MDP) theory.

**Q84: What is "meta-reinforcement learning"?**
A: Meta-RL trains an agent to "learn to learn" — adapt to new tasks quickly. The agent is trained on a distribution of tasks. Each "meta-episode" consists of multiple RL episodes for a single task. The agent learns a policy that can be adapted (via internal memory or gradient updates) to new tasks within a few episodes. Architectures: 1) RL² (Duan et al., 2016) — RNN-based agent that learns a learning algorithm in its hidden state. 2) MAML-RL (Finn et al., 2017) — learn initial parameters that can adapt quickly with a few gradient steps. Meta-RL enables few-shot adaptation, but requires a good task distribution for training.

**Q85: What is "offline reinforcement learning"?**
A: Offline RL (batch RL) learns a policy from a fixed dataset without any environment interaction. The agent must learn from logged data as if it were doing RL, but cannot explore. Key challenge: distribution shift — the learned policy may take actions not present in the dataset, and the Q-function may incorrectly estimate their values (extrapolation error). Approaches: 1) Policy constraints (BCQ, BEAR, TD3+BC) — constrain the policy to stay close to the data distribution. 2) Value regularization (CQL) — penalize overestimation of out-of-distribution actions. 3) Implicit Q-learning (IQL) — avoid querying out-of-distribution actions entirely.

**Q86: Explain "Conservative Q-Learning" (CQL).**
A: CQL (Kumar et al., 2020) addresses the overestimation problem in offline RL by learning a conservative Q-function. The CQL regularizer: minimize Q(s,a) for actions in the learned policy (push down), maximize Q(s,a) for actions in the dataset (push up). This prevents the Q-function from overestimating unseen actions. CQL can be added to any Q-learning objective: L = L_TD + α * (E_{s∼D, a∼π}[Q(s,a)] - E_{s∼D, a∼D}[Q(s,a)]). CQL is one of the most effective offline RL algorithms, achieving strong results on D4RL benchmarks.

**Q87: What is "TD3+BC" for offline RL?**
A: TD3+BC (Fujimoto & Gu, 2021) is a simple modification of TD3 for offline RL. It adds a behavioral cloning (BC) regularization term to the policy update: π = argmax_π E[Q(s, π(s)) - α * (π(s) - a)^2]. The BC term keeps the policy close to actions in the dataset. Key insight: the Q-function should only be maximized for actions near the data distribution. TD3+BC is remarkably simple (minimal code change from TD3) and competitive with more complex offline RL methods like CQL. The α parameter controls how tightly the policy stays near the dataset.

**Q88: Explain "risk-sensitive RL".**
A: Risk-sensitive RL considers not just expected return but also the distribution of returns (risk). Approaches: 1) Worst-case criterion — maximize minimum possible return (robust). 2) Conditional Value at Risk (CVaR) — maximize expected return in the worst α-percentile of outcomes. 3) Variance-penalized — maximize E[R] - λ Var(R). 4) Distributional RL — learn full return distribution, then use risk measures to select actions. 5) Entropy-regularized — SAC implicitly manages risk through entropy. Risk-sensitive RL is important for safety-critical applications (finance, healthcare, autonomous driving).

**Q89: What is "safety in reinforcement learning"?**
A: Safe RL addresses the problem of learning policies that both maximize reward and satisfy safety constraints. Approaches: 1) Constrained MDP (CMDP) — maximize reward subject to cost constraints. 2) Lagrangian methods — primal-dual optimization for constrained RL (e.g., PPO-Lagrangian, SAC-Lagrangian). 3) Safety layers — project unsafe actions to safe alternatives. 4) Shielding — use formal methods to guarantee safety during learning. 5) Lyapunov-based — use Lyapunov functions to guarantee stability. 6) Reward shaping — penalize unsafe behavior. Safe RL is critical for real-world deployment.

**Q90: What is "transfer learning" in RL?**
A: Transfer learning in RL reuses knowledge from a source task to accelerate learning on a target task. Types: 1) Representation transfer — share feature representations (network weights). 2) Policy transfer — directly transfer or fine-tune the policy. 3) Value transfer — transfer value function estimates. 4) Model transfer — transfer environment dynamics model. 5) Option transfer — transfer discovered sub-policies. Challenges: 1) Negative transfer — source task harms target task learning. 2) Mismatch — different state/action spaces require mapping. 3) Catastrophic forgetting — agent forgets source task when learning target. Domain randomization is related — trains on many variations for robust zero-shot transfer.

## 7. Practical RL & Real-World Deployment (Q91–Q100)

**Q91: What are the main RL libraries and frameworks available?**
A: 1) Stable-Baselines3 (SB3) — reliable implementations of major RL algorithms in PyTorch (PPO, SAC, TD3, DQN, A2C). 2) Ray RLlib — scalable RL library for distributed training, supports multi-agent, custom models. 3) Acme (DeepMind) — modular RL framework for research. 4) Tianshou — lightweight PyTorch RL library with clean code. 5) Dopamine (Google) — research-focused DQN variants. 6) CleanRL — single-file implementations for readability. 7) Sample Factory — high-performance RL for games. 8) Gymnasium (previously OpenAI Gym) — standard RL environment interface.

**Q92: How do you evaluate an RL agent's performance?**
A: Key metrics: 1) Average return — mean episode reward over evaluation episodes. 2) Success rate — percentage of episodes achieving the goal. 3) Sample efficiency — performance vs environment steps. 4) Wall-clock time — real time to reach a performance level. 5) Robustness — performance across random seeds, environment variations. 6) Learning curve — performance versus training steps. 7) Regret — cumulative difference from optimal performance. 8) Safety metrics — constraint violations, failure rate. Evaluation protocols: run multiple random seeds (5-10), report mean and variance, use fixed evaluation policy.

**Q93: What are common RL environments and benchmarks?**
A: 1) Gymnasium (MuJoCo, Atari, Box2D) — standard continuous and discrete control benchmarks. 2) D4RL — offline RL benchmarks (MuJoCo, AntMaze, kitchen, adroit). 3) Procgen — procedurally generated environments for generalization testing. 4) DM Control Suite — continuous control from DeepMind. 5) NetHack — challenging dungeon-crawling game. 6) SMAC (StarCraft Multi-Agent Challenge) — MARL benchmark. 7) Meta-World — multi-task continuous control. 8) Isaac Gym — GPU-accelerated physics simulation. 9) Safety Gym — safe RL benchmarks. 10) MineRL — Minecraft-based RL.

**Q94: How do you handle the reproducibility challenge in RL?**
A: Reproducibility is difficult in RL due to: stochasticity in environments and policies, hardware-dependent randomness, and hyperparameter sensitivity. Best practices: 1) Fix random seeds for environment, agent, and network initialization. 2) Report results over multiple seeds (at least 5). 3) Use deterministic operations where possible (torch.use_deterministic_algorithms). 4) Record all hyperparameters. 5) Use established benchmarks and evaluation protocols. 6) Log evaluation runs separately from training runs (no training noise in evaluation). 7) Version-control code and environment configurations.

**Q95: What is the "RL-to-real" gap and how is it addressed?**
A: The RL-to-real gap is the difference between simulation and real-world performance. Causes: 1) Simulator inaccuracies — physics, visuals, dynamics differ from reality. 2) Latency — simulators run fast, real systems have latency. 3) Sensor noise — simulators have idealized sensors. 4) Actuator limitations — real motors have wear, heat, non-linearities. Mitigations: 1) Domain randomization — randomize simulator parameters during training (friction, mass, color, lighting). 2) System identification — calibrate simulator to match real system. 3) Robust control — train policies that are robust to perturbations. 4) Sim-to-real transfer — train in simulation, fine-tune on real system. 5) Real-world RL — learn directly on real systems (requires safety measures).

**Q96: How is RL applied in robotics?**
A: RL applications in robotics: 1) Locomotion — walking, running, jumping (legged robots). 2) Manipulation — grasping, pushing, assembly (robot arms). 3) Navigation — mobile robots, drone flight. 4) Dexterous manipulation — in-hand object manipulation. 5) Soft robotics — controlling deformable robots. Key techniques for robotics RL: 1) Simulation-based training with domain randomization. 2) Curriculum learning — progressive task difficulty. 3) Reward shaping — guide learning with intermediate rewards. 4) Imitation + RL — pre-train from demonstrations, then fine-tune with RL. 5) Safe exploration — use safety shields in real-world deployment.

**Q97: How is RL used in recommendation systems and advertising?**
A: RL in recommendation: the recommender is the agent, user interactions form the environment, and engagement metrics (clicks, dwell time, purchases) are rewards. Challenges: 1) Large action spaces — millions of items. 2) Delayed rewards — user may purchase days after recommendation. 3) Non-stationarity — user preferences change. 4) Exploration vs exploitation — balance showing known-good recommendations vs discovering new ones. Approaches: 1) Contextual bandits — simple RL for single-step recommendations. 2) DQN/Deep RL for session-based recommendations. 3) Slate-based RL — recommend multiple items at once. Industry use: YouTube, Netflix, Google Ads, Meta advertising.

**Q98: Explain how RL is applied in autonomous driving.**
A: RL in autonomous driving: 1) Motion planning — learn driving policies (steering, acceleration, braking). 2) Traffic negotiation — merge, cross intersections, handle roundabouts. 3) Lane changing — when and how to change lanes safely. 4) Energy optimization — for electric vehicles. Challenges: 1) Safety-critical — mistakes are dangerous. 2) Rare events — difficult to learn from experience alone. 3) Multi-agent — other drivers, pedestrians are also learning/adapting. 4) Regulation — must be verifiably safe. Solutions: 1) Simulation-first with massive domain randomization. 2) Imitation learning from human drivers + RL fine-tuning. 3) Constrained RL for safety. 4) Hierarchical RL for maneuver-level planning.

**Q99: What are the computational challenges in scaling RL?**
A: 1) Experience collection — environment interaction is the bottleneck (especially real-world). 2) Model capacity — deep networks for high-dimensional observations require significant compute. 3) Batch size — some algorithms (PPO, SAC) benefit from large batches for stable updates. 4) Multi-agent scaling — joint action spaces grow exponentially. 5) Long horizons — long episodes need more memory and compute. 6) Parallelism — need many environment copies for efficient training. Infrastructure: 1) Distributed RL frameworks (Ray RLlib, Sample Factory, SEED RL). 2) GPU-accelerated simulators (Isaac Gym, DM-Visual). 3) Cloud compute for large-scale experiments.

**Q100: What are the current limitations and future directions of RL?**
A: Current limitations: 1) Sample inefficiency — most RL algorithms require millions of interactions. 2) Generalization — policies fail when environment changes slightly. 3) Reward specification — designing good reward functions is hard. 4) Safety — unsafe exploration in real-world deployment. 5) Reproducibility — results vary across seeds and implementations. 6) Evaluation — no standard benchmarks for generalization, safety, robustness. Future directions: 1) Foundation models for RL — large pre-trained models as policy backbones. 2) RL + LLMs — combining reasoning with RL (RLHF, Reflexion). 3) World models — learning environment models for efficient planning. 4) Offline RL — learning from static datasets without interaction. 5) Multi-agent RL for complex systems — traffic, economy, climate. 6) Safe RL — formal guarantees for real-world deployment. 7) Meta-learning — agents that learn to learn across tasks. 8) Embodied RL — robots learning in the wild.
