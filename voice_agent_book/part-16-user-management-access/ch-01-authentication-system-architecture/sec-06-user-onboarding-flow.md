# Section 06: User Onboarding Flow

User onboarding guides new users through their first experience with the platform. The goal is to activate the user—get them to complete their first meaningful action (creating an agent, making a test call). Onboarding is personalized based on the user's role, plan tier, and use case.

Onboarding steps: welcome screen (product overview, value proposition), profile completion (name, company, role), use case selection (customer support, outbound sales, appointment scheduling, healthcare), first agent creation (guided wizard: name, prompt, voice selection), test call (make a sample call to test the agent), and integration setup (connect phone number, configure webhook). Progress is saved; users can skip and return later.

Onboarding automation: behavior tracking identifies where users drop off (e.g., 60% complete agent creation but don't test call). Automated emails send tips and encouragement for unfinished steps. In-app help (tooltips, guided tours, contextual documentation) is shown during onboarding. Completion rate is tracked as a key product metric.
