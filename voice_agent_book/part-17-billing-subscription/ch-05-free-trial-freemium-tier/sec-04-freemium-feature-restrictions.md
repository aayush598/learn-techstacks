# Section 04: Freemium Feature Restrictions

## Feature Limits on Free Tier

The Free tier (Freemium) provides enough value to demonstrate the product while creating natural upgrade pressure. Feature restrictions are designed to be frustrating enough to motivate upgrades but not so restrictive that users abandon the product entirely.

```typescript
interface FreemiumRestriction {
  feature: string;
  restrictionType: 'limit' | 'watermark' | 'delay' | 'hide' | 'degrade';
  limitValue?: number;
  upgradePrompt: {
    enabled: boolean;
    frequency: 'once' | 'daily' | 'always';
    message: string;
  };
```

## Watermarking

Watermarking adds visible indicators to feature outputs to remind free users they're on a limited plan. Common watermarks include branding on exported files, "Powered by" tags on call transcripts, and delayed data availability.

```typescript
class WatermarkService {
  applyWatermark(
    content: string,
    feature: string,
    source: string
  ): string {
    switch (feature) {
      case 'transcript_export':
        return this.watermarkTranscript(content);
      case 'call_recording':
        return this.watermarkRecording(content);
      case 'analytics_export':
        return this.watermarkAnalytics(content);
      default:
        return content;
    }
  }

  private watermarkTranscript(transcript: string): string {
    return `${transcript}\n\n---\nGenerated by Voice Agent Platform (Free Tier)\nUpgrade to remove watermark: https://platform.com/upgrade`;
  }

  private watermarkAnalytics(content: string): string {
    return `[PREVIEW] ${content}\n\nFull analytics available on paid plans.\nUpgrade at: https://platform.com/upgrade`;
  }

  shouldWatermark(planId: string, feature: string): boolean {
    return planId === 'free' && FREEMIUM_RESTRICTIONS.some(
      r => r.feature === feature && r.restrictionType === 'watermark'
    );
  }
}
```

## Delayed Analytics

Free users get delayed access to analytics data. While paid users see real-time data, free users see data with a 24-hour delay. This creates a clear value differentiator while still providing useful information.

```typescript
class AnalyticsDelayService {
  async getAnalyticsData(
    tenantId: string,
    query: AnalyticsQuery
  ): Promise<AnalyticsResult> {
    const plan = await this.planService.getTenantPlan(tenantId);
    const restriction = FREEMIUM_RESTRICTIONS.find(
      r => r.feature === 'analytics_data'
    );

    if (plan.id === 'free' && restriction) {
      // Apply delay: only show data older than 24 hours
      const delayMs = restriction.limitValue * 3600000;
      const cutoff = new Date(Date.now() - delayMs);

      query.endDate = cutoff.toISOString();
    }

    return this.analyticsEngine.query(tenantId, query);
  }
}
```

## Upgrade Prompts

Strategic upgrade prompts are shown at natural moments: when hitting usage limits, when accessing premium features, or after completing key actions. Prompts should be contextual and helpful, not aggressive.

```typescript
class UpgradePromptService {
  async shouldShowPrompt(
    tenantId: string,
    context: PromptContext
  ): Promise<PromptResult> {
    const plan = await this.planService.getTenantPlan(tenantId);
    if (plan.id !== 'free') return { show: false };

    const restriction = FREEMIUM_RESTRICTIONS.find(
      r => r.feature === context.feature
    );
    if (!restriction || !restriction.upgradePrompt.enabled) {
      return { show: false };
    }

    // Check frequency
    if (restriction.upgradePrompt.frequency !== 'always') {
      const lastPrompt = await this.db.upgradePrompts.findOne({
        tenantId,
        feature: context.feature,
      }, { sort: { shownAt: -1 } });

      if (lastPrompt) {
        if (restriction.upgradePrompt.frequency === 'once') {
          return { show: false };
        }
        if (restriction.upgradePrompt.frequency === 'daily') {
          const lastPromptDay = new Date(lastPrompt.shownAt).toDateString();
          if (lastPromptDay === new Date().toDateString()) {
            return { show: false };
          }
        }
      }
    }

    // Show prompt
    await this.db.upgradePrompts.create({
      tenantId,
      feature: context.feature,
      shownAt: new Date().toISOString(),
    });

    return {
      show: true,
      message: restriction.upgradePrompt.message.replace(
        '{{percent}}',
        Math.round(context.currentUsage / context.limit * 100).toString()
      ),
      upgradeUrl: `${APP_URL}/pricing?source=${context.feature}`,
    };
  }
}
```

## Open-Source Tools

- **Unleash** (Apache 2.0) — Feature flag-based restriction management
- **PostgreSQL** — Freemium configuration and user state
- **Redis** — Rate limit enforcement for free tier
- **BullMQ** — Schedule restriction evaluation jobs

## Integration Points

Freemium restrictions connect to the feature gate service (runtime enforcement), the API middleware (rate limiting), the analytics service (data delay), and the notification service (upgrade prompts).

## Production Considerations

- Monitor free-to-paid conversion rate by feature
- A/B test different restriction levels
- Track feature usage on free tier to identify upgrade triggers
- Avoid making free tier so painful that users churn entirely
- Review restriction strategy quarterly based on conversion data

## Open-Source First Philosophy

Unleash (Apache 2.0) provides the feature flag infrastructure for freemium restrictions without proprietary software. PostgreSQL stores restriction configuration in version-controlled YAML files. This open-source approach replaces expensive feature management platforms while providing fine-grained control over free tier features.
