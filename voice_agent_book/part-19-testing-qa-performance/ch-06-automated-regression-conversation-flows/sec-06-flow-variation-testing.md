# Section 06: Flow Variation Testing

## Overview

Flow variation testing validates that agents handle different conversational paths correctly, not just the happy path. Each flow definition includes multiple variation dimensions: user personality types, emotional states, phrasing styles, and unexpected inputs. Variation testing ensures agents are robust across diverse user interactions.

Test variations are generated by combining different parameters: user cooperativeness (helpful, confused, impatient), utterance complexity (simple, detailed, rambling), emotional tone (neutral, frustrated, happy), and environmental factors (background noise, poor connection).

## Design Decisions

- **React Flow over Custom**: Battle-tested node/edge rendering. Saves 6+ months custom development.
- **Local-First State**: Zustand + debounced saves (2s). Instant UI response without waiting for API.
- **Function-as-Edge**: Edges carry conditions and transforms. Flow evaluates conditions at each step.
## Implementation Approach

```typescript
interface FlowVariation {
  persona: 'helpful' | 'confused' | 'impatient' | 'angry' | 'distracted';
  complexity: 'simple' | 'moderate' | 'complex';
  emotionalTone: 'neutral' | 'frustrated' | 'happy' | 'anxious';
  environment: 'quiet' | 'noisy' | 'echoing' | 'low-bandwidth';
}

class FlowVariationTester {
  async testVariations(flow: FlowDefinition, config: VariationConfig): Promise<VariationReport> {
    const results: VariationResult[] = [];
    
    // Generate variation combinations
    const combinations = this.generateCombinations(config);
    
    for (const variation of combinations) {
      // Configure utterance generator for this variation
      const generator = new UtteranceGenerator({
        persona: variation.persona,
        complexity: variation.complexity,
        emotionalTone: variation.emotionalTone,
      });
      
      // Run simulation with this variation
      const engine = new SimulationEngine(generator);
      const result = await engine.simulate(flow.name, {
        agentId: config.agentId,
        variations: [variation],
      });
      
      results.push({
        variation,
        passed: result.passed,
        path: result.path,
        metrics: result.metrics,
        issues: result.issues,
      });
    }
    
    return {
      flowName: flow.name,
      totalVariations: combinations.length,
      passedVariations: results.filter(r => r.passed).length,
      failedVariations: results.filter(r => !r.passed).length,
      details: results,
      coverage: this.calculateCoverage(results, config),
      recommendations: this.generateRecommendations(results),
    };
  }

  private generateCombinations(config: VariationConfig): FlowVariation[] {
    const combos: FlowVariation[] = [];
    
    for (const persona of config.personas) {
      for (const complexity of config.complexities) {
        for (const tone of config.emotionalTones) {
          combos.push({ persona, complexity, emotionalTone: tone, environment: 'quiet' });
        }
      }
    }
    
    // Add environmental variations for key combinations
    for (const env of config.environments) {
      combos.push({
        persona: 'helpful',
        complexity: 'moderate',
        emotionalTone: 'neutral',
        environment: env,
      });
    }
    
    return combos;
  }
}
```

## Integration Points

- **Simulation Engine**: Variations feed into utterance generation
- **Regression Suite**: Variation tests part of regression suite
- **Coverage Dashboard**: Variation coverage tracked
- **Performance Testing**: Performance baselines per variation
- **Quality Scoring**: Variation pass rate contributes to overall quality score

## Open-Source Tools

- **React Flow** (MIT): Node-based UI
- **Zustand** (MIT): State management
- **Immer** (MIT): Immutable updates
## Production Considerations

- **Combinatorial Explosion**: Full combination sets grow exponentially; use smart sampling
- **Test Duration**: Each variation adds test time; prioritize critical variations
- **Result Complexity**: Many variations produce complex results; invest in visualization
- **Persona Maintenance**: User personas need updating based on real user data
