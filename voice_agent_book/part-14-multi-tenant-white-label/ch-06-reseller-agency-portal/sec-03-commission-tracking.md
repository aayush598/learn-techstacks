# Section 03: Commission Tracking System

## Overview

The commission tracking system calculates and manages reseller commissions based on their sub-accounts' subscription revenue. Commissions are the primary incentive for resellers—they earn a percentage of the revenue generated by the tenants they bring to the platform. The system tracks commissionable events (new subscriptions, renewals, upgrades), calculates commissions based on tier-based rates, manages payout schedules, and provides dispute resolution.

Commission calculation considers the reseller's tier (higher tiers earn higher commission rates), the type of revenue (new business vs renewal, monthly vs annual), and any promotions or discounts applied. Commissions may be one-time (for annual contracts) or recurring (for monthly subscriptions). The system supports holdbacks (retaining a percentage until certain conditions are met) and clawbacks (recovering commissions on early cancellations).

For a voice agent platform, commissions typically range from 10-30% of monthly recurring revenue, with higher rates for new business acquisition. The commission structure must be competitive enough to attract quality resellers while maintaining platform profitability.

## Implementation Approach

```typescript
interface CommissionRule {
  id: string;
  resellerTierId: string;
  revenueType: 'new' | 'renewal' | 'upgrade';
  billingPeriod: 'monthly' | 'annual';
  rate: number; // Percentage
  holdbackPercent: number;
  clawbackPeriodDays: number;
}

class CommissionCalculator {
  async calculateEarnedCommissions(
    resellerId: string,
    period: DateRange
  ): Promise<CommissionSummary> {
    const reseller = await this.getReseller(resellerId);
    const rules = await this.getCommissionRules(reseller.tierId);
    const invoices = await this.getSubAccountInvoices(resellerId, period);

    let totalCommission = 0;
    const lineItems: CommissionLineItem[] = [];

    for (const invoice of invoices) {
      const rule = this.findMatchingRule(rules, invoice);
      const commissionAmount = invoice.amount * (rule.rate / 100);
      const holdback = commissionAmount * (rule.holdbackPercent / 100);
      const netCommission = commissionAmount - holdback;

      lineItems.push({
        invoiceId: invoice.id,
        subAccountId: invoice.tenantId,
        revenueAmount: invoice.amount,
        commissionRate: rule.rate,
        grossCommission: commissionAmount,
        holdback,
        netCommission,
      });

      totalCommission += netCommission;
    }

    return {
      resellerId,
      period,
      totalCommission,
      lineItems,
      status: 'calculated',
    };
  }

  async processPayout(resellerId: string, period: DateRange): Promise<Payout> {
    const commissions = await this.calculateEarnedCommissions(resellerId, period);
    
    if (commissions.totalCommission <= 0) return null;

    // Apply minimum payout threshold
    if (commissions.totalCommission < 50) {
      await this.rollToNextPeriod(resellerId, period);
      return null;
    }

    // Create payout via Stripe Connect
    const payout = await this.stripe.transfers.create({
      amount: Math.round(commissions.totalCommission * 100), // cents
      currency: 'usd',
      destination: reseller.stripeAccountId,
      transfer_group: `commission-${period.start.toISOString()}`,
    });

    // Record payout
    await this.db.query(`
      INSERT INTO commission_payouts (reseller_id, period_start, period_end, 
        amount, stripe_transfer_id, status, created_at)
      VALUES ($1, $2, $3, $4, $5, 'completed', NOW())
    `, [resellerId, period.start, period.end, commissions.totalCommission, payout.id]);

    return {
      id: payout.id,
      amount: commissions.totalCommission,
      currency: 'usd',
      lineItems: commissions.lineItems,
    };
  }

  async handleClawback(subAccountId: string, cancellationDate: Date): Promise<void> {
    // Find commissions paid for this sub-account within the clawback period
    const recentCommissions = await this.db.query(`
      SELECT * FROM commission_payouts cp
      JOIN commission_line_items cl ON cp.id = cl.payout_id
      WHERE cl.sub_account_id = $1
        AND cp.created_at > $2 - INTERVAL '90 days'
    `, [subAccountId, cancellationDate]);

    for (const commission of recentCommissions.rows) {
      // Reverse the commission
      await this.stripe.transfers.createReversal(commission.stripe_transfer_id);
      
      // Log clawback
      await this.auditService.log('commission.clawback', {
        subAccountId,
        amount: commission.amount,
        reason: 'early_cancellation',
      });
    }
  }

  async getResellerCommissionReport(resellerId: string, year: number): Promise<Report> {
    return this.db.query(`
      SELECT 
        DATE_TRUNC('month', created_at) as month,
        SUM(amount) as commissions,
        COUNT(*) as transactions
      FROM commission_payouts
      WHERE reseller_id = $1 AND EXTRACT(YEAR FROM created_at) = $2
      GROUP BY DATE_TRUNC('month', created_at)
      ORDER BY month
    `, [resellerId, year]);
  }
}
```

## Production Considerations

- **Payout Schedule:** Process commissions monthly, with a reconciliation period (e.g., pay on the 15th of the following month). Allow manual payout triggering for special cases.
- **Minimum Payout Threshold:** Set a minimum payout amount ($50) to avoid processing fees eating into small commissions. Roll over amounts below threshold to next period.
- **Clawback Period:** Standard clawback period is 90 days. If a customer cancels within 90 days, the commission is reversed. Track clawbacks and allow resellers to dispute.
- **Tax Reporting:** Issue 1099-NEC forms in the US for resellers earning over $600/year. Integrate with tax reporting services (Stripe Tax, TaxJar).
- **Commission Disputes:** Provide a dispute resolution process. Resellers can challenge clawbacks with evidence of customer satisfaction.
- **Audit Trail:** All commission calculations, payouts, and adjustments are logged immutably. This is critical for reseller trust and legal compliance.
