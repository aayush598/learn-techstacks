# Section 08: Security Patch Compliance Reporting

Security patch compliance reporting provides visibility into patch status across all systems. Reports show: systems requiring patches, patch deployment status, time-to-patch metrics, SLA compliance, and exceptions. Reports are generated weekly for operational review and monthly for compliance evidence.

Report metrics: patch coverage (percentage of systems with latest security patches within SLA), mean time to patch (MTTP, from CVE publication to production deployment), patch failure rate (percentage of patches that failed or required rollback), overdue patches (count by severity, by system), and exception inventory (active exceptions with expiry dates). Trend charts show improvement or degradation over time.

Report distribution: weekly patch report → engineering team (actionable items), monthly compliance report → security team (SOC 2 evidence), quarterly executive report → management (risk overview, trends). The compliance dashboard provides real-time patch status visualization. Automated notifications for: new critical patches requiring action, patch SLAs approaching breach, and patches that failed deployment.
