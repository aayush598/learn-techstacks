# Section 08: Security Incident Response Plan

The Security Incident Response Plan (SIRP) defines the process for detecting, responding to, and recovering from security incidents. The plan covers all incident types: data breach, service compromise, unauthorized access, malware, insider threat, and denial of service. The plan is tested quarterly through tabletop exercises and annual full-scale simulations.

Incident response team roles: Incident Commander (coordinates response, makes decisions), Security Lead (technical investigation, containment), Communications Lead (internal/external communications, regulatory notifications), Legal Counsel (legal obligations, liability assessment), and Engineering Lead (service restoration, patching). Roles are defined with backup assignments.

Response phases: Preparation (runbooks, tools, training), Detection & Analysis (alert → initial assessment → classification → escalation), Containment, Eradication & Recovery (isolate affected systems, remove threat, restore clean state), Post-Incident Activity (root cause analysis, lessons learned, control improvements). Communication templates are pre-written for: internal team notifications, executive summaries, customer notifications (breach), and regulatory authority filings.
