# Section 03: Type Generation from OpenAPI

## Overview

TypeScript types for the SDK are automatically generated from the OpenAPI specification using openapi-typescript. This ensures type safety across the entire API surface — request parameters, response bodies, and error shapes are all derived from the single source of truth. The generation pipeline runs as part of the CI/CD process.

## Design Decisions

- **openapi-typescript Over Manual Types**: Auto-generated types never drift from API spec; manual types require maintenance
- **Custom Type Overrides**: Some types need refinement (e.g., datetime strings → Date objects); use type overrides
- **Partial Type Generation**: Generate per-resource types independently for tree-shaking
- **Spec as Source of Truth**: API spec is authoritative; Zod schemas generate the spec, spec generates types

## Implementation Approach

```typescript
// package.json scripts
{
  "scripts": {
    "generate:types": "openapi-typescript ./spec/api.yaml -o ./src/generated/types.ts",
    "generate:types:watch": "openapi-typescript ./spec/api.yaml -o ./src/generated/types.ts --watch"
  }
}

// Generated type usage
import type { components, paths } from '../generated/types';

// Extract specific types
type Agent = components['schemas']['Agent'];
type CreateAgentRequest = components['schemas']['CreateAgentRequest'];
type AgentListResponse = paths['/v1/agents']['get']['responses']['200']['content']['application/json'];

// Type overrides for refinement
interface AgentOverrides {
  createdAt: Date;
  updatedAt: Date;
}

// Custom type with overrides applied
type AgentWithDates = Omit<Agent, 'createdAt' | 'updatedAt'> & AgentOverrides;

// Response type mapper
class ResponseTypeMapper {
  mapAgent(raw: components['schemas']['Agent']): AgentWithDates {
    return {
      ...raw,
      createdAt: new Date(raw.createdAt),
      updatedAt: new Date(raw.updatedAt),
    };
  }

  mapAgentList(raw: AgentListResponse): ListResponse<AgentWithDates> {
    return {
      data: raw.data.map(a => this.mapAgent(a)),
      pagination: raw.pagination,
    };
  }
}

// SDK resource uses generated types
class AgentsResource {
  constructor(
    private http: HttpClient,
    private mapper: ResponseTypeMapper,
  ) {}

  async list(params?: {
    cursor?: string;
    limit?: number;
    status?: string;
  }): Promise<ListResponse<AgentWithDates>> {
    const raw = await this.http.get<AgentListResponse>('/v1/agents', { params });
    return this.mapper.mapAgentList(raw);
  }

  async get(id: string): Promise<AgentWithDates> {
    const raw = await this.http.get<paths['/v1/agents/{id}']['get']['responses']['200']['content']['application/json']>(
      `/v1/agents/${id}`,
    );
    return this.mapper.mapAgent(raw);
  }

  async create(data: CreateAgentRequest): Promise<AgentWithDates> {
    const raw = await this.http.post<paths['/v1/agents']['post']['responses']['201']['content']['application/json']>(
      '/v1/agents',
      { body: data },
    );
    return this.mapper.mapAgent(raw);
  }
}
```

## Integration Points

- **CI/CD**: Type generation runs on spec changes; PRs fail if generated types have breaking changes
- **Monorepo**: Generated types shared across SDK, documentation, and internal tools
- **OpenAPI Diff**: Breaking changes in spec trigger type generation failure

## Production Considerations

- **Type Generation Speed**: For large specs, generation takes ~5 seconds; configure CI caching
- **Custom Type Overrides**: Maintain overrides file that's updated when spec changes
- **Version Alignment**: SDK version matches API version; types generated from matching spec version
- **Backward Compatibility**: Additive spec changes (new optional fields) don't break existing types

## Open-Source Tools

- **openapi-typescript**: Generate TypeScript types from OpenAPI spec
- **@hey-api/openapi-ts**: Alternative type generator with runtime client generation
