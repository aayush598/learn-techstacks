# Section 07: Optimization Insights and Recommendations

## Overview

Optimization insights and recommendations transform funnel analysis data into actionable suggestions for improving conversion rates and reducing drop-offs. The system analyzes funnel metrics, drop-off patterns, and comparison results to generate prioritized recommendations. Each recommendation includes: the affected stage, the estimated impact (improvement in conversion rate), the effort level required (low/medium/high), a suggested action, and references to supporting data.

Recommendations are generated by a rule-based engine that encodes known best practices for contact center optimization. Examples: "IVR drop-off rate is 25% — consider reducing menu options from 5 to 3" (based on research showing 20% improvement with simplified IVR), "Queue abandonment is high during peak hours (2-4 PM) — consider dynamic staffing adjustments," "Transfer rate from Tier 1 to Tier 2 is 40% — review Tier 1 agent training on handling complex issues." The engine also identifies custom patterns from historical data: "When the queue depth exceeds 10, abandonment rate increases by 15% — consider queue depth alert at 8."

## Architecture

```
           Optimization Insights Pipeline

   ClickHouse (funnel metrics, drop-offs, comparisons)
        |
   Insights Engine
        |
   ┌────┴────────────┐
   |                 |
   Rule-Based        Pattern-Based
   (best practices)  (historical correlation)
   |                 |
   ClickHouse        Recommendation
   (insight store)   Prioritizer
                     (impact × confidence)
   |
   Dashboard (insight cards,
   estimated impact, effort level,
   "apply" action button)
```

## Design Decisions

- **Rule-based + pattern-based hybrid over pure ML for recommendations:** Rule-based recommendations capture established best practices and are immediately actionable. Pattern-based recommendations (discovered from correlation analysis in the tenant's own data) provide custom insights specific to the tenant's operations. A pure ML approach (e.g., recommending actions by learning from successful interventions) would require historical intervention data that most tenants don't have. Trade-off: the rule set must be maintained by domain experts; rules that become outdated may suggest suboptimal actions.

- **Impact estimation using historical benchmark data over theoretical models:** When the system recommends "reduce IVR options from 5 to 3," it estimates impact based on aggregated benchmark data from other tenants who made similar changes. Impact is expressed as a range ("10-20% reduction in IVR drop-off") with confidence level. The benchmark database is populated from anonymized, aggregated data across all tenants (with opt-out option). Trade-off: cross-tenant benchmarks may not apply directly to a specific tenant's context; displayed as "estimated" with a "your results may vary" note.

- **Actionable "quick apply" links over insight-only display:** Each recommendation includes a button or link that navigates directly to the relevant configuration page (IVR editor, staffing scheduler, agent training module) with suggested values pre-filled. Reducing the friction between insight and action increases the likelihood that recommendations are implemented. Trade-off: "quick apply" requires integration with the configuration systems (IVR builder, WFM, training platform), which may not all be available as APIs.

## Implementation Approach

```typescript
interface OptimizationRecommendation {
  id: string;
  tenantId: string;
  funnelId: string;
  stageId: string;
  category: 'ivr' | 'queue' | 'routing' | 'agent' | 'process' | 'self_service';
  title: string;
  description: string;
  currentMetric: { name: string; value: number; threshold: number };
  estimatedImpact: {
    metric: string;
    improvement: string;            // e.g., "+15% conversion"
    confidence: 'high' | 'medium' | 'low';
    range: [number, number];       // e.g., [10, 20] percent improvement
  };
  effort: 'low' | 'medium' | 'high';
  priority: number;                  // 1 (highest) - 100
  suggestedAction: {
    type: 'navigate' | 'configure' | 'contact' | 'learn';
    target: string;                  // URL or action identifier
    label: string;
  };
  supportingData: Array<{
    label: string;
    value: string;
    visual?: 'gauge' | 'sparkline' | 'number';
  }>;
  dismissed: boolean;
  createdAt: number;
}

class OptimizationInsightEngine {
  private clickhouse: ClickHouseClient;

  async generateRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    const recommendations: OptimizationRecommendation[] = [];

    // IVR recommendations
    const ivrRecs = await this.generateIvrRecommendations(tenantId, funnelId);
    recommendations.push(...ivrRecs);

    // Queue recommendations
    const queueRecs = await this.generateQueueRecommendations(tenantId, funnelId);
    recommendations.push(...queueRecs);

    // Routing recommendations
    const routingRecs = await this.generateRoutingRecommendations(tenantId, funnelId);
    recommendations.push(...routingRecs);

    // Agent performance recommendations
    const agentRecs = await this.generateAgentRecommendations(tenantId, funnelId);
    recommendations.push(...agentRecs);

    // Process recommendations
    const processRecs = await this.generateProcessRecommendations(tenantId, funnelId);
    recommendations.push(...processRecs);

    // Prioritize
    return this.prioritizeRecommendations(recommendations);
  }

  private async generateIvrRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    const recs: OptimizationRecommendation[] = [];

    // Check IVR drop-off rate
    const ivrDropOff = await this.clickhouse.query(`
      SELECT
        countIf(eventType = 'stage_entry') as entries,
        countIf(eventType = 'stage_dropoff') as dropOffs
      FROM funnel_events
      WHERE tenantId = '${tenantId}'
        AND funnelId = '${funnelId}'
        AND stageId = 'ivr'
        AND timestamp >= now() - INTERVAL 7 DAY
    `);

    if (ivrDropOff.length > 0) {
      const dropOffRate = ivrDropOff[0].entries > 0
        ? (ivrDropOff[0].dropOffs / ivrDropOff[0].entries) * 100
        : 0;

      if (dropOffRate > 15) {
        recs.push({
          id: `ivr_dropoff_${tenantId}`,
          tenantId,
          funnelId,
          stageId: 'ivr',
          category: 'ivr',
          title: 'High IVR drop-off rate',
          description: `${dropOffRate.toFixed(1)}% of callers abandon during IVR. Consider reducing menu options, adding speech recognition, or offering a callback option.`,
          currentMetric: { name: 'IVR Drop-off Rate', value: dropOffRate, threshold: 15 },
          estimatedImpact: {
            metric: 'IVR Completion Rate',
            improvement: 'Expected 10-20% reduction in IVR drop-off',
            confidence: 'high',
            range: [10, 20],
          },
          effort: 'medium',
          priority: 0,
          suggestedAction: {
            type: 'navigate',
            target: `/ivr-editor/${funnelId}`,
            label: 'Edit IVR Flow',
          },
          supportingData: [
            { label: 'Current', value: `${dropOffRate.toFixed(1)}%`, visual: 'number' },
            { label: 'Industry Benchmark', value: '10%', visual: 'number' },
          ],
          dismissed: false,
          createdAt: Date.now(),
        });
      }
    }

    return recs;
  }

  private async generateQueueRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    const recs: OptimizationRecommendation[] = [];
    const now = Date.now();
    const peakHours = [14, 15, 16]; // 2-4 PM

    // Check queue abandonment during peak hours
    const queueAbandon = await this.clickhouse.query(`
      SELECT
        countIf(eventType = 'stage_abandon') as abandons,
        countIf(eventType = 'stage_entry') as entries
      FROM funnel_events
      WHERE tenantId = '${tenantId}'
        AND funnelId = '${funnelId}'
        AND stageId = 'queue'
        AND toHour(toDateTime(timestamp / 1000)) IN (${peakHours.join(',')})
        AND timestamp >= ${now - 30 * 24 * 3600 * 1000}
    `);

    if (queueAbandon.length > 0) {
      const peakAbandonRate = queueAbandon[0].entries > 0
        ? (queueAbandon[0].abandons / queueAbandon[0].entries) * 100
        : 0;

      if (peakAbandonRate > 20) {
        recs.push({
          id: `queue_peak_abandon_${tenantId}`,
          tenantId,
          funnelId,
          stageId: 'queue',
          category: 'queue',
          title: 'High queue abandonment during peak hours',
          description: `${peakAbandonRate.toFixed(1)}% of callers abandon the queue between 2-4 PM. Consider adjusting staff schedules or enabling callback during peak hours.`,
          currentMetric: { name: 'Peak Hour Abandonment', value: peakAbandonRate, threshold: 20 },
          estimatedImpact: {
            metric: 'Queue Conversion',
            improvement: 'Expected 15-25% reduction in queue abandonment',
            confidence: 'medium',
            range: [15, 25],
          },
          effort: 'high',
          priority: 0,
          suggestedAction: {
            type: 'navigate',
            target: '/staffing/schedules',
            label: 'Adjust Staffing Schedule',
          },
          supportingData: [
            { label: 'Peak Hour Abandonment', value: `${peakAbandonRate.toFixed(1)}%`, visual: 'gauge' },
            { label: 'Off-Peak Abandonment', value: '8%', visual: 'number' },
          ],
          dismissed: false,
          createdAt: Date.now(),
        });
      }
    }

    return recs;
  }

  private async generateRoutingRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    const recs: OptimizationRecommendation[] = [];

    // Check transfer rate
    const transferRate = await this.clickhouse.query(`
      SELECT
        countIf(eventType = 'stage_entry' AND previousStageId = 'agent_tier1') as transfers,
        countIf(eventType = 'stage_entry' AND stageId = 'agent_tier1') as entries
      FROM funnel_events
      WHERE tenantId = '${tenantId}'
        AND funnelId = '${funnelId}'
        AND timestamp >= now() - INTERVAL 30 DAY
    `);

    if (transferRate.length > 0) {
      const tRate = transferRate[0].entries > 0
        ? (transferRate[0].transfers / transferRate[0].entries) * 100
        : 0;

      if (tRate > 30) {
        recs.push({
          id: `high_transfer_${tenantId}`,
          tenantId,
          funnelId,
          stageId: 'agent_tier1',
          category: 'routing',
          title: 'High Tier 1 to Tier 2 transfer rate',
          description: `${tRate.toFixed(1)}% of calls are transferred from Tier 1 to Tier 2. Review Tier 1 training on handling complex issues and consider skill-based routing.`,
          currentMetric: { name: 'Transfer Rate', value: tRate, threshold: 30 },
          estimatedImpact: {
            metric: 'First Call Resolution',
            improvement: 'Expected 10-15% improvement in FCR',
            confidence: 'medium',
            range: [10, 15],
          },
          effort: 'medium',
          priority: 0,
          suggestedAction: {
            type: 'navigate',
            target: '/routing/skill-based',
            label: 'Review Routing Rules',
          },
          supportingData: [
            { label: 'Transfer Rate', value: `${tRate.toFixed(1)}%`, visual: 'gauge' },
            { label: 'Target', value: '< 30%', visual: 'number' },
          ],
          dismissed: false,
          createdAt: Date.now(),
        });
      }
    }

    return recs;
  }

  private async generateAgentRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    // Identify agents with below-average conversion rates at the resolution stage
    const agentPerf = await this.clickhouse.query(`
      SELECT
        agentId,
        a.name as agentName,
        countIf(eventType = 'stage_entry') as entries,
        countIf(eventType = 'stage_exit') as exits
      FROM funnel_events fe
      JOIN agents a ON fe.agentId = a.id
      WHERE fe.tenantId = '${tenantId}'
        AND fe.funnelId = '${funnelId}'
        AND fe.stageId = 'resolution'
        AND fe.timestamp >= now() - INTERVAL 30 DAY
      GROUP BY agentId, a.name
      HAVING entries >= 20
      ORDER BY (exits / entries) ASC
      LIMIT 3
    `);

    // Would generate recommendations for lowest-performing agents
    return [];
  }

  private async generateProcessRecommendations(
    tenantId: string,
    funnelId: string
  ): Promise<OptimizationRecommendation[]> {
    // Check for excessive time in any stage
    const longStages = await this.clickhouse.query(`
      SELECT
        stageId,
        avg(durationMs) as avgDuration,
        countIf(durationMs > 600000) as longCalls
      FROM funnel_events
      WHERE tenantId = '${tenantId}'
        AND funnelId = '${funnelId}'
        AND eventType = 'stage_exit'
        AND timestamp >= now() - INTERVAL 7 DAY
      GROUP BY stageId
      HAVING avgDuration > 300000
    `);

    return longStages.map((stage: any) => ({
      id: `long_stage_${stage.stageId}_${tenantId}`,
      tenantId,
      funnelId,
      stageId: stage.stageId,
      category: 'process' as const,
      title: `Excessive time in ${stage.stageId} stage`,
      description: `Average time in ${stage.stageId} is ${(stage.avgDuration / 60000).toFixed(1)} minutes. ${stage.longCalls} calls exceeded 10 minutes. Review the process flow for inefficiencies.`,
      currentMetric: { name: 'Avg Stage Duration', value: stage.avgDuration / 60000, threshold: 5 },
      estimatedImpact: {
        metric: 'Stage Duration',
        improvement: 'Expected 20-30% reduction',
        confidence: 'medium' as const,
        range: [20, 30],
      },
      effort: 'high' as const,
      priority: 0,
      suggestedAction: {
        type: 'navigate',
        target: `/process/${stage.stageId}/review`,
        label: 'Review Process',
      },
      supportingData: [
        { label: 'Average', value: `${(stage.avgDuration / 60000).toFixed(1)} min`, visual: 'number' as const },
        { label: 'Calls > 10 min', value: stage.longCalls.toString(), visual: 'number' as const },
      ],
      dismissed: false,
      createdAt: Date.now(),
    }));
  }

  private prioritizeRecommendations(
    recs: OptimizationRecommendation[]
  ): OptimizationRecommendation[] {
    const effortWeight: Record<string, number> = { low: 1, medium: 0.5, high: 0.2 };
    const confidenceWeight: Record<string, number> = { high: 1, medium: 0.5, low: 0.2 };

    return recs
      .map(r => {
        const impactScore = (r.estimatedImpact.range[0] + r.estimatedImpact.range[1]) / 2;
        const priority = impactScore * confidenceWeight[r.estimatedImpact.confidence] * effortWeight[r.effort];
        return { ...r, priority: Math.round(priority * 100) };
      })
      .sort((a, b) => b.priority - a.priority);
  }
}

// Recommendation card component
const InsightCard: React.FC<{
  recommendation: OptimizationRecommendation;
  onDismiss: (id: string) => void;
}> = ({ recommendation: rec, onDismiss }) => (
  <div className={`insight-card priority-${rec.priority > 50 ? 'high' : rec.priority > 20 ? 'medium' : 'low'}`}>
    <div className="insight-header">
      <CategoryBadge category={rec.category} />
      <EffortBadge effort={rec.effort} />
      <PriorityScore score={rec.priority} />
    </div>
    <h4>{rec.title}</h4>
    <p>{rec.description}</p>
    <div className="insight-impact">
      <span className="impact-badge">
        {rec.estimatedImpact.improvement}
      </span>
      <span className="confidence">{rec.estimatedImpact.confidence} confidence</span>
    </div>
    <div className="insight-actions">
      <Button onClick={() => window.location.href = rec.suggestedAction.target}>
        {rec.suggestedAction.label}
      </Button>
      <Button variant="ghost" onClick={() => onDismiss(rec.id)}>Dismiss</Button>
    </div>
  </div>
);
```

## Production Considerations

**Scaling:** The insight engine runs nightly for each tenant and generates recommendations that are stored in ClickHouse. Each tenant's analysis processes the last 30-90 days of funnel events — for large tenants (1M+ events/month), this completes in under 30 seconds. Recommendations are cached in Redis with a 24-hour TTL (refreshed nightly). Users can manually trigger a refresh if they want up-to-date results after making changes.

**Security:** Recommendations are tenant-scoped. Agent-specific recommendations (e.g., "Agent Smith has below-average conversion") require `agent-performance:view` permission. The benchmark data used for impact estimation is aggregated across tenants and does not expose individual tenant data — tenants can opt out of contributing to benchmarks. Recommendation dismissal is per-user (each supervisor can dismiss different recommendations).

**Monitoring:** Track recommendation generation rate, average priority score, recommendation click-through rate (how often users click "apply"), and recommendation dismissal rate. Alert if the recommendation engine produces no recommendations for 7 consecutive days (may indicate data pipeline issue). Track the most common recommendation categories to guide product improvements.
