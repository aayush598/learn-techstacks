# Time Management During Interview

## The Clock in Your Head

Interview time management is about pacing, not rushing.

## The 5-5-20-10-5 Rule

| Interval | Focus |
|----------|-------|
| 0-5 min | Understand, clarify, examples |
| 5-10 min | Brute force + optimization discussion |
| 10-30 min | Code the optimal solution |
| 30-40 min | Test, debug, handle edge cases |
| 40-45 min | Complexity, trade-offs, follow-ups |

## What If You Get Stuck?

**Don't panic**. Use these strategies:

1. **Go to brute force**: "Let me at least get a working solution first."
2. **Simplify**: "What if I assume all inputs are positive/sorted/unique?"
3. **Think out loud**: "I'm stuck because X. Let me try a different angle..."
4. **Trace examples**: "Let me manually trace through this example to spot the pattern."
5. **Ask for hint**: "I'm considering two approaches but not sure which to pursue."

## When You Can't Finish Coding

1. **Summarize approach**: "I would complete the function by adding X..."
2. **Explain remaining logic**: "The for loop would check each neighbor..."
3. **State complexities**: "The finished solution would be O(n) time..."
4. **Acknowledge trade-offs**: "If I had more time, I'd optimize by..."

## Handling Follow-ups

When interviewer asks a harder follow-up:
- "Let me think about this extension."
- "This adds complexity because now we need X."
- "I can solve this by modifying our approach to..."

## Managing Anxiety

- **Breathe**: Take 3 deep breaths before starting
- **Water**: Sip water to create thinking pauses
- **Positive self-talk**: "I've solved problems like this before."
- **Focus on the problem, not the outcome**

## Practice With a Timer

Use LeetCode's built-in timer or a separate stopwatch:
- Easy: 10-15 min
- Medium: 25-30 min
- Hard: 35-45 min

If you exceed these consistently, you need more focused practice on that topic.
